Responsive Web Design

Fluid Design

Modelo que utiliza porcentajes para definir el tamaño de los elementos de la pagina y su relacion entre si. Uno de los problemas del mismo radica en que nuestro diseño pierde proporcion en pantallas pequeñas, ya que los porcentajes se mantienen; y por otro lado, en pantallas grandes se estira demasiado el contenido.
Una alternativa a este tipo de diseño es el Fluid Design que aplica atributos de maximos y minimos para evitar la compresion y estiramiento de los elementos. El problema radica en los espacios generados para la separacion de contenido.

Adaptive Design

Modelo que implementa el diseño individual de contenido para cada tamaño de pantalla, facilita la implementacion del diseño y es una alternativa para clientes con poco presupuesto. Se utilizan herramientas de redireccionamiento para que el cliente visite la version mas adecuada para su dispositivo.
Uno de los problemas principales de este tipo de contenido radica en la duplicidad del mismo y la repeticion de archivos dentro del sitio, ya que se usa uno distinto segun el tamaño de pantalla para que se diseñe.

Responsive Design

Modelo cuyos origenes radican en una publicacion realizada por Ethan Marcotte en el sitio A List Apart donde explica como se deberian implementar los Media Quieries, el diseño a partir de la vista en dispositivos moviles y el concepto conocido como One Web for All donde debemos utilizar una unica fuente de informacion (estructura).
Una evolucion de este tipo de diseño es la que se conoce como Content-first, ya que a diferencia del Mobile-first, plantea empezar a trabajar con el contenido final.
