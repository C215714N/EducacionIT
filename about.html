Formularios HTML

Son herramientas que permiten la interaccion (intercambio de datos) entre cliente y servidor. Para implementarlos dentro de una pagina web debemos utilizar la etiqueta <form> y solo se considerara para el envio, los datos que se encuentren dentro del mismo, ademas se deben especificar algunas caracteristicas para que este se realice correctamente.

¿Como funcionan?

A la etiqueta mencionada anteriormente, tenemos que especificarle 2 atributos, por un lado action="..." que indica al formulario que hacer al momento de realizar el envio y method="..." que indica quien tiene la prioridad al momento de realizar el mismo, dentro de los metodos existentes podemos utilizar:

  • GET: metodo poco seguro, que prioriza la respuesta del servidor, los datos permanecen visibles en la url y se utiliza principalmente para buscadores
  • POST: metodo seguro que cifra los datos antes y despues del envio, debido a esto es mas lento, pero ofrece cierta garantia para proteger los datos del usuario. Se utiliza para formularios de registro, acceso y consulta web

¿Como estan Compuestos?

Dentro de un formulario utilizaremos varios elementos, pero entre ellos destacan las Etiquetas (LABEL) que funcionan como rotulos para indicar al usuario que debe seleccionar o completar; otro elemento necesario son los CAMPOS (INPUT / TEXTAREA / SELECT) que permiten al usuario la carga o seleccion de datos, estos poseen atributos que en ciertas ocasiones deben estar escritos de manera explicita.

  • name="..." corresponde al nombre del grupo de campos al que pertenece el control, necesario para el envio.
  • value="..." indica el dato a ser enviado, no es necesario si el usuario puede escribir el valor.
  • type="..." corresponde al formato del dato a ingresar/seleccionar, es necesario para los campos de tipo INPUT.

Contenido Multimedia

Desde sus comienzos se pueden utilizar este tipo de elementos dentro de la web, pero debido a las limitaciones de la epoca era poco comun el uso de imagenes y mucho menos videos. Antes de la version 5 de html, tambien era necesario la implementacion de plugins o complementos de terceros para reproducir dicho contenido, e incluso se utilizaban etiquetas para incorporar dicho contenido de manera asincrona.

¿Contenido Embebido?

Corresponde a una categoria de la web, que indica a los motores de busqueda que el contenido utilizado dentro de nuestra web fue compartido por el autor del mismo. Normalmente implementamos esto utilizando la etiqueta <iframe> (Marco Flotante) que permite la carga asincronica de contenido, es decir, que no afecta a los tiempos de carga de nuestra pagina.

¿Multimedia sin plugins?

Como se menciona en la seccion anterior, antiguamente se necesitaban plugins como flash player para reproducir este tipo de contenido dentro de la pagina. En la actualidad los navegadores incluyen una API de reproduccion de contenido y para acceder a esta debemos utilizar las etiquetas <audio> y <video>

Si bien este tipo de contenido se reproduce de manera nativa, debemos tener en cuenta los formatos compatibles, que son .mp3, .wav y .ogg para audio y .mp4, .webm y .ogv para video; ademas si queremos que se muestren los controles en pantalla tendremos que utilizar el atributo controls.

Tablas HTML

Son elementos que permiten organizar la informacion de manera tabular, que deben utilizarse cuando los datos requiren respetar cierta estructura. Para incorporarlas dentro de la pagina debemos utilizar la etiqueta <table> y dentro colocar los elementos que corresponden a la misma (filas y celdas).

¿Cual es su estructura?

En el interior de la etiqueta de tabla debemos colocar las etiquetas <tr> (Table Row) para indicar las filas contenedoras y dentro de las mismas las celdas que pueden ser de encabezado <th> (Table Heading) o de contenido <td> (Table Division). Ademas de estos elementos, debemos especificar la semantica de la tabla utilizando los siguientes elementos.

  • THEAD Cabecera de tabla que contiene los encabezados que representan los datos
  • TBODY Cuerpo de tabla que contiene los datos y generalmente posee varias filas
  • TFOOT Pie de tabla que contiene informacion complementaria a los datos
