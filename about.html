Formularios HTML

Son herramientas que permiten el intercambio de datos entre cliente y servidor, para implementarlos dentro de la pagina debemos colocar la etiqueta <form> y dentro de la misma los elementos que seran procesados para el envio. Cabe destacar que, por defecto, hasta que el usuario no presione un boton que ejecute dicha accion, ningun dato sera enviado al servidor; tampoco se consideraran elementos que no se encuentran dentro de la etiqueta.

¿Como funcionan?

A la etiqueta mencionada anteriormente debemos especificarle por lo menos 2 atributos, estos son action="..." que indica que debe suceder cuando el usuario realiza el envio del formulario; y method="..." el cual indica como debe realizarse dicho envio, entre los valores que podemos utilizar se encuentran:

  • GET metodo que prioriza la respuesta del servidor, muestra los datos enviados en la URL y los envia en texto plano. Este metodo es ideal para buscadores.
  • POST metodo cuya prioridad es garantizar la seguridad de los datos enviados por el usuario, debido a que cifra los mismos antes de realizar el envio. Este metodo se utiliza en mayoria de formularios como contacto, registro, inicio de sesion, etc....

¿Como estan compuestos?

Dentro de un formulario podemos colocar cualquier elemento HTML existente, pero existen 2 elementos necesarios, estos son las etiquetas (LABEL) que funcionan a modo de rotulos, indicando al usuario lo que debe seleccionar o completar; y los Campos (INPUT / TEXTAREA / SELECT) que corresponden a los controles donde el usuario debe ingresar/seleccionar los datos.

  • name identificar los campos de formulario que corresponden a la misma opcion
  • type identifica el tipo de valor que se debe colocar dentro del campo, obligatorio para los input.
  • value corresponde al dato que sera enviado, es obligatorio en caso que el usuario seleccione.

Contenido Multimedia

Se identifica con este nombre a todo elemento audiovisual que podemos utilizar dentro de una pagina web. Antiguamente para poder implementar este tipo de elemtos era necesaria la instalacion de algun plugin o complemento de terceros, debido a que los navegadores no brindaban soporte nativo. Pero con la ultima actualizacion de HTML, los navegadores incluyen APIs de reproduccion de contenido.

¿Sin Plugins?

Como se menciona anteriormente, los navegadores soportan este tipo de elementos de forma nativa, gracias a la API de reproduccion, que podemos utilizar con las etiquetas <audio> y <video>; siendo compatible con los formatos .mp3, .wav, .ogg y .mp4, .webm, .ogv respectivamente. Ademas podemos especificar atributos como controls, autoplay, muted, loop, etc... segun sea necesario.
