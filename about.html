Formularios HTML

Son herramientas que permiten el intercambio de datos entre cliente y servidor, para poder implementarlos debemos colocar dentro de la pagina la etiqueta <form> y solo se considerara para el envio de los datos, lo que se encuentre dentro del mismo. Ademas de dicha etiqueta debemos especificar ciertos atributos y colocar determinados elementos que se detallan en este articulo.

多Como funcionan?

A la etiqueta mencionada anteriormente debemos colocarle por lo menos 2 atributos, action="..." que indica lo que debe suceder al momento de realizar el envio del formulario, en la actualidad se suele utilizar un script en lenguaje de servidor, como por ejemplo PHP; y method="..." que indica como se procesa el envio y de quien es la prioridad en el intercambio de datos, los valores mas comunes son:

  • GET: Metodo mas rapido que prioriza la respuesta del servidor, la informacion siempre permanece visible en la barra de direcciones y envia los datos en texto plano. Se suele utilizar para Buscadores (Consulta Query)
  • POST: Metodo mas seguro, debido a que encripta los datos antes de realizar el envio, estos no figuran en ningun otro lado que no sea el formulario, pero es mas lento. Se suele utilizar para Formularios de Registro, Acceso (Inicio sesion), Contacto (Consulta Web)

多Como se componen?

Si bien en los formularios podemos colocar cualquiera de las etiquetas vistas anteriormente, vale aclarar que si no se realiza el envio, los datos no se almacenan en ningun lado. Tambien es necesario trabajar con dos elementos que guian al usuario a la hora de completar el mismo. Entre estos elementos encontramos:

  • Etiquetas (LABEL): Rotulo que indica al usuario cual es el valor a ingresar/seleccionar, facilitan la accesibilidad y deben estar vinculados con algun campo del formulario.
  • Campos (INPUT/TEXTAREA/SELECT): Controles que permiten al usuario seleccionar/escribir los datos solicitados por el formulario, estos poseen 2 atributos que son type="..." indicando el formato de los datos y value="..." indicando cual es el dato seleccionado/ingresado.

Contenido Multimedia

Desde un comienzo, la carga de este tipo de contenido era poco comun debido al ancho de banda manejado por las conexiones, pero con el paso del tiempo fue incorporandose en mayor medida. En un principio era necesario instalar algun complemento o plugin como flash player, pero actualmente los navegadores cuentan con una api de reproduccion nativa.

多Contenido Embebido?

Es una categoria que se utiliza para identificar a todo contenido que haya sido compartido desde la web del autor y utilizado con permiso del mismo en otro sitio; asi como tambien a los elementos que se utilizan dentro de la pagina web, que no pertecen al universo HTML. En la mayoria de los casos haremos uso de la etiqueta <iframe> que permite la carga asincrona de contenido en nuestra pagina.

多Multimedia sin plugins?

Como se menciono anteriormente, era muy comun el uso de plugins para reproducir contenido multimedia, pero el inconveniente de esto radicaba en que el usuario necesitaba instalar un complemento que en ocasiones no era compatible con algun navegador. Para incorporar este tipo de elementos debemos utilizar las etiquetas <audio> y <video> que son compatibles con los formatos .mp3, .wav, .ogg y .mp4, .webm, .ogv respectivamente, cabe destacar que para mostrar los controles del reproductor es necesario utilizar el atributo controls

Tablas HTML

Son herramientas que permiten organizar la informacion de manera tabular y tenemos que utilizarlas cuando los datos a representar requieren de una estructura que no debe ser alterada. Al igual que en cualquier otro ambito, las mismas se componen de filas, columnas y celdas pero en HTML se crean respetando el modelo de cajas, utilizando la etiqueta <table> como contenedor principal

多Cual es su estructura?

Las tablas, a diferencia de otras representaciones en programas, no poseen columnas definidas, es decir que dentro del contenedor principal utilizaremos etiquetas como <tr> (table row) para indicar al contenedor de fila y <td> (table division) para indicar la celda de contenido. Ademas poseen estructura semantica y podemos clasificar su partes utilizando los siguientes contenedores:

  • THEAD: Cabecera de tabla, contiene los encabezados que se representan con la etiqueta <th> (table heading)
  • TBODY: Cuerpo de tabla, contiene los datos representados y si realizamos tabla de referencias cruzadas tambien puede poseer encabezados.
  • TFOOT: Pie de tabla, contiene datos complementarios al cuerpo, como por ejemplo: totales, promedios, maximos, minimos, etc...

多Como se combinan las celdas?

En ciertas ocasiones necesitaremos que una celda ocupe mas de una fila o una columna, para ello existe el atributo rowspan="..." y colspan="..." indicando como valor la cantidad final de celdas que deben utilizarse, cabe descatar que las celdas correlativas seran desplazadas hacia la derecha segun el espacio ocupado.
