Formularios HTML

Son herramientas que permiten el intercambio de datos entre cliente y servidor, para implementarlos dentro de nuestras paginas debemos utilizar la etiqueta <form> y solamente se procesaran para el envio, los datos que se encuentren dentro de los elementos de la misma. A diferencia de otros comportamientos de la pagina, la repuesta brindada por el servidor variara segun el tipo de formulario que utilicemos.

多Como Funcionan?

Dentro de la etiqueta mencionada anteriormente debemos colocar al menos 2 atributos, action="..." que determina que sucedera con los datos una vez realizado el envio y donde generalmente se utiliza un script en lenguaje de backend (ej: php); y method="..." que indica como se debe procesar el envio de los datos, entre los valores mas comunes se encuentran:

  • GET: Metodo cuya prioridad radica en la respuesta del servidor, es menos confiable ya que los datos figuran en la barra de direcciones y se envian en texto plano, pero es mucho mas rapido. Los formularios que utilizan este metodo son los Buscadores
  • POST: metodo cuya prioridad radica en la seguridad de los datos del usuario, ya que cifra los mismos antes de realizar el envio y la informacion no se visualiza en la barra de direcciones, por eso es mas lento. Los formularios que utilizan estos metodos son Registro, Acceso y Contacto

多Cual es su Estructura?

Dentro de un formulario podemos encontrar varias etiquetas, pero entre ellas se destacan 2 tipos de elementos, los Rotulos (Label) que se utilizan para guiar al usuario al momento de completar el formulario; y los Campos (INPUT / TEXTAREA / SELECT) que corresponden a los elementos que permiten seleccionar o ingresar datos y poseen las siguientes caracteristicas:

  • name: corresponde al nombre o grupo de pertenencia de la etiqueta, que permite identificar al elemento en cuestion.
  • type: corresponde al tipo de datos permitido, como texto, numeros, fecha etc... con sus respectivas restricciones.
  • value: corresponde al valor ingresado o seleccionado, en este ultimo caso debe encontrarse en el campo de manera explicita.

Contenido Multimedia

Son elementos graficos y audiovisuales que podemos utilizar dentro de nuestra pagina. Antiguamente era necesario utilizar plugins o herramientas de terceros para incorporarlos, especialmente en el caso de los elementos de tipo video, pero los navegadores actuales los soportan de manera nativa. Aun asi todavia se siguen implementando herramientas que permiten la carga asincrona de este tipo de contenido.

多Contenido Embebido?

Corresponde a una categoria web que indica que dicho contenido fue compartido con autorizacion del autor original. Este tipo de contenido se incorpora normalmente utilizando la etiqueta <iframe> que permite mostrar contenido de otra web en una ventana flotante, sin afectar los tiempos de carga.

多Multimedia sin Plugins?

Como se menciona mas arriba, actualmente los navegadores soportan este tipo de contenido de forma nativa, gracias a la API de reproduccion que utiliza las etiquetas <audio> y <video> que son compatibles con los formatos .mp3, .wav, .ogg y .mp4, .webm, .ogv respectivamente.
