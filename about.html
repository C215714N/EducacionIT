Formularios HTML

Son herramientas que permiten el intercambio de datos entre cliente y servidor, para incorporarlos dentro de la pagina debemos colocar la etiqueta <form> y dentro de la misma los elementos que seran procesados para el envio.

¿Como funcionan?

A la etiqueta mencionada anteriormente debemos colocarle por al menos 2 atributos, estos son action="..." que indica lo que debe suceder al realizar el envio del mismo, utilizandose normalmente un script en lenguaje de servidor (ej: PHP); y method="..." que especifica como se debe realizar el intercambio de datos, utilizando alguno de los valores predefinidos.

  • GET: Metodo que prioriza la respuesta del servidor, envia los datos en texto plano y estos permanecen visibles en la url. Se utilizan principalmente en formularios de busqueda.
  • POST: Metodo cuya prioridad el la seguridad de los datos, ya que cifra los mismos antes de realizar el envio. Se utiliza en formularios de Registro, Acceso, Contacto, etc...

¿Como estan compuestos?

Dentro de un formulario, podemos colocar cualquier etiqueta existente, pero destacan 2 tipos de elementos, las Etiquetas (LABEL) que se utilizar para guiar al usuario al momento de completar el formulario; y los Campos (INPUT / TEXTAREA / SELECT) que corresponden a los controles del mismo y seran utilizados por el usuario. los campos poseen los siguientes atributos segun su tipo.

  • name="..." Nombre con el que se identifica al campo de formulario, es necesario para la captura de datos y debe ser compartido en caso que varios campos correspondan a la misma opcion.
  • type="..." corresponde al tipo de dato que se debe utilizar, en el caso del INPUT es obligatorio en caso de requerir un valor distinto a texto plano, mientras que en los otros campos es implicito.
  • value="..." Es el dato en si mismo y usualmente es definido por el usuario cuando escribe en el formulario. En caso de seleccionar debe ser explicito dentro de los atributos del campo.

Contenido Multimedia

Termino que corresponde a los elementos audiovisuales que podemos utilizar en la pagina. Actualmente los navegadores brindan soporte nativo a los mismos y sin necesidad de instalar algun complemento o plugins de terceros (ej: flash player). Podemos corroborar la compatibilidad de nuestro navegador con estos y cualquier otro elemento desde el sitio Can I Use

¿Contenido Embebido?

Se conoce con este nombre a una categoria web que indica que un elemento fue compartido con autorizacion del publicador original. En la direccion o su contenido encontraremos la palabra embed y los podremos incorporar gracias a la etiqueta <iframe> que permite la carga asincrona de contenido.

¿Multimedia sin Plugins?

Como se hace referencia al principio los navegadores incluyen una API de reproduccion a la que podemos acceder si utilizamos las etiquetas <audio> o <video> que son compatibles con los formatos .mp3, .wav, .ogg y .mp4, .webm, .ogv respectivamente. Tambien podemos utilizar atributos como autoplay, muted, loop y controls

Tablas HTML

Son herramientas que permiten organizar la informacion de manera tabular y deben ser utilizadas cuando los datos se corresponden con cierta estructura. Para incorporalas dentro de la pagina simplemente debemos colocar la etiqueta <table> y dentro de la misma los elementos que la componen, respetando el modelo de cajas.

¿Poseen Estructura Semantica?

Dentro de la etiqueta mencionada anteriormente, debemos colocar las filas utilizando la etiqueta <tr>(table row) y dentro de estas las celdas de datos con la etiqueta <td>(table division). Pero a partir de HTML5 se incorporaron elementos que permiten a los motores de busqueda identificar las partes de la tabla:

  • THEAD: Cabecera de tabla que contiene las celdas de encabezado, las cuales se agregan utilizando la etiqueta <th> (table heading).
  • TBODY: Cuerpo de tabla que contienen los datos relevantes para el usuario, e incluso puede llegar a incorporar celdas de encabezado.
  • TFOOT: Pie de tabla que corresponde a los datos complementarios de la misma, como por ejemplo totales, promedios, maximos, minimos, etc...
