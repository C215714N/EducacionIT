Formularios HTML

Son herramientas que permiten el intercambio de datos entre cliente y servidor, ademas son los unicos elementos de la pagina que permiten al usuario intervenir directamente sobre la pagina. Para implementarlos debemos utilizar la etiqueta <form> y dentro de la misma colocar los elementos que seran utilizados para realizar una peticion al servidor.

多como funcionan?

A la etiqueta mencionada anteriormente se le deben especificar por lo menos 2 atributos, entre ellos se encuentran action="..." que se utiliza para indicar lo que sucedera al momento de realizar el envio del formulario, para lo que normalmente se utiliza un archivo en lenguaje de servidor como PHP; y luego esta method="..." el cual se utiliza para indicar como se realizara el envio y la prioridad del mensaje, entre los metodos mas comunes, se encuentran los siguientes.

  • GET metodo rapido que prioriza los resultados de la consulta, no encripta los datos y la informacion utilizada en el formulario permanece visible en todo momento. Se utiliza principalmenta para formularios de Busqueda (Consulta Query)
  • POST metodo seguro que prioriza los datos del usuario y por lo tanto, encripta la informacion antes de realizar el envio, es mas lento y admite un mayor caudal de datos. Se utiliza para formularios de Registro - Acceso (Inicio Sesion) - Contacto.

多como estan compuestos?

Si bien, en un formulario podemos colocar cualquiera de las etiquetas utilizadas anteriormente, en los mismos existen 2 elementos clave para el correcto funcionamiento del mismo, estos se clasifican de la siguiente manera.

  • Etiquetas (LABEL) Textos que funcionan a modo de rotulos, indicando al usuario que debe completar dentro del campo de formulario con el que esta asociado, tambien facilitan la accesibilidad, ya que representan el texto legible.
  • Campos (INPUT / TEXTAREA / SELECT) Elementos donde el usuario selecciona o escribe sus datos, poseen atributos como value="..." que indica que datos se va a enviar en la consulta y type="..." que indica el formato de los mismos.

Contenido Multimedia

Al principio no era posible subir este tipo de contenido a un sitio, debido a que el ancho de banda de las conexiones de internet no era suficiente como para cargar video o audio, incluso era dificil la incorporacion de imagenes sin que afecten los tiempos de carga de la pagina. Tampoco habia compatibilidad nativa para este tipo de elementos y por lo tanto era necesario la instalacion de plugins o complementos de terceros para poder reproducirlos, los cuales no siempre eran compatibles con el navegador del usuario.

多Contenido Embebido?

Una de las formas de incorporar contenido multimedia (principalmente video) es mediante una categoria de HTML conocida como embed (embebido) que se utiliza para identificar un contenido que fue compartido con autorizacion del autor de la web original, e incluso mostar contenido que no pertenece al universo HTML. Normalmente se incorpora mediante la etiqueta <iframe> que permite la carga asincrona de datos, es decir, que los mismos no afectan los tiempos de carga de nuestro sitio

多Multimedia sin plugins?

Otra forma de incorporar contenido original a nuestra web es mediante las etiquetas <audio> y <video> que invocan a la API de reproduccion del navegador y los formatos compatibles con estas son .mp3 - .wav - .ogg y .mp4 - .webm - .ogv respectivamente. Cabe destacar que solamente apareceran los controles en pantalla si utilizamos el atributo controls e incluso podemos habilitar la reproduccion automatica con atributos como autoplay muted y loop.
