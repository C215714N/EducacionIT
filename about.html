Formularios HTML

Son herramientas que permiten la interaccion (intercambio de datos) entre cliente y servidor, para poder implementarlos dentro de una pagina web debemos utilizar la etiqueta <form>, ademas de especificar algunas caracteristicas. Solamente se tomara en cuenta, al momento del envio, los contenidos que se encuentren dentro de la etiqueta; es decir, que todo elemento fuera del formulario no sera considerado parte del mismo.

¿Como funcionan?

A la etiqueta mencionada anteriormente debemos especificarle por lo menos dos atributos, estos son action="..." que indica lo que debe ocurrir al momento de realizar el envio y method="..." que indica quien tiene la prioridad y como se deben procesar los datos, entre los valores mas comunes se encuentran:

  • GET: Metodo que prioriza la respuesta del servidor, los datos se envian en texto plano, permanecen visibles en la barra de direcciones y se utilizan principalmente para los buscadores.
  • POST: Metodo que prioriza la seguridad de los datos del usuario, ya que los mismos se cifran entre el envio y la recepcion, se utilizan para formularios de registro, acceso, contacto, etc...

¿Como estan compuestos?

Dentro de un formulario, ademas de las etiquetas que ya conocemos, encontraremos dos elementos necesarios; por un lado estan las Etiquetas (LABEL) que indican al usuario lo que debe seleccionar o completar, y por otro los Campos (INPUT / SELECT / TEXTAREA) donde el usuario completa o selecciona datos, a la vez estos campos poseen los siguientes atributos:

  • name="..." indica el grupo de pertenencia dentro de las opciones del formulario.
  • value="..." corresponde al dato ingresado por el usuario, en caso de seleccionar opciones debe estar de manera explicita.
  • type="..." equivale al formato de los datos a ingresar, si corresponde con algun numero, fecha o cadena de texto.

Contenido Multimedia

Corresponde a todo elemento grafico y audiovisual que podemos utilizar dentro de una pagina web. Antiguamente eran poco frecuentes debido a las limitaciones de ancho de banda, pero en la actualidad se puede incorporar para estilizar el sitio web, mostrar contenido sin necesidad de texto y ademas los navegadores los admiten de forma nativa.

¿Contenido Embebido?

Categoria que se utiliza para identificar que un elemento fue compartido con autorizacion del autor o publicador original. Se suele incorporar mediante la etiqueta <iframe> que permite la carga de contenido de manera asincrona.

¿Multimedia Sin Plugins?

Como se menciona anteriormente, hoy en dia no es necesario instalar ningun plugin o complemento en el navegador, ya que estos incorporan una API de reproduccion de contenido. Para acceder a estas debemos utilizar las etiquetas <audio> y <video> que son compatibles con los formatos .mp3, .wav, .ogg y .mp4, .webm, .ogv respectivamente.

Tablas HTML

Es una estructura de datos que se utiliza para organizar la informacion de manera tabular, podemos implementarlas dentro de una pagina web colocando la etiqueta <table> y dentro de la misma las filas con <tr> (Table Row), asi como <td>(Table division) y <th> (Table Heading) para las celdas de contenido y los encabezados.

¿Poseen Estructura Semantica?

La estructura anteriormente mencionada debe crearse respetando el modelo de cajas, es decir que posee elementos contenedores y de contenido y a partir de HTML5 se incorporaron nuevas etiquetas que facilitan tanto a los desarrolladores como a los motores de busqueda la comprension de las mismas, estas son:

  • THEAD: Cabecera de tabla que contiene las celdas de encabezado.
  • TBODY: Cuerpo de tabla que contiene los datos a representar.
  • TFOOT: Pie de tabla que contiene datos complementarios.

¿Expansion de Filas y Columnas?

En ciertas ocasiones sera necesario que alguna de las celdas de nuestras tablas deba ocupar mas de una columna o fila, para ello utilizaremos los atributos colspan="..." y rowspan="..." respectivamente para expandir el tamaño de las mismas. Cabe destacar que todo contenido que se encuentre en la posicion ocupada se desplazara hacia la derecha.
