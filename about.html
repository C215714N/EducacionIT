Formularios HTML

Son herramientas que permiten el intercambio de datos entre cliente y servidor, siendo el unico lugar de la pagina donde el usuario puede intervenir directamente. Para poder implementarlos debemos colocar la etiqueta <form> y dentro de la misma, los elementos que seran procesados al realizar el envio del mismo.

多Como funcionan?

  • GET: metodo rapido que prioriza la respuesta del servidor, envia los datos en texto plano y estos permanecen visibles en la URL al realizar el mismo. Debido a esto ultimo, se lo considera un metodo poco seguro y solamente se lo utiliza para formularios de Busqueda (Consulta Query)
  • POST: metodo que prioriza la seguridad de los datos del cliente, ya que encripta los mismos antes de realizar el envio del formulario. Debido a esto es mas lento, pero funciona como medida de seguridad ante el robo de datos y se lo utiliza par la mayoria de los formularios como por ejemplo: Registro, Acceso, Contacto (Consulta Web).

多Como se componen?

  • Etiqueta (LABEL) Corresponde al rotulo que identifica que tipo dato debemos seleccionar o completar en el formulario. Facilita la accesiblidad y debe vincularse con al menos un campo.
  • Campos (INPUT/TEXTAREA/BUTTON) corresponden a los elementos sobre los cuales el usuario puede escribir o seleccionar alguna opcion, estos poseen 2 atributos importantes que son type="..." que indica el tipo de dato que se de ingresar/seleccionar y value="..." que corresponde al dato ingresado/seleccionado.

Tablas HTML

Son herramientas que permiten organizar la informacion de manera tabular y deben utilizarse siempre y cuando los datos deban manetener determinada estructura. A diferencia de otros ambitos, las mismas se crean respetando el modelo de cajas en el cual existen elementos contenedores y de contenido.

多Cual es su estructura?

Las tablas se componen de filas, columnas y celdas, pero se representan con las etiquetas <table> como contenedor principal, <tr> (table row) como contenedor de fila y por ultimo <td> (table division) como celda de contenido. Ademas con la aparicion de la ultima version de html se incorporo una estructura semantica con los siguientes elementos:

  • THEAD: Cabecera de tabla, se utiliza para contener a los encabezados que se representan con la etiqueta th (table heading)
  • TBODY: Cuerpo de tabla, es aquel que contiene los datos representados y en ocasiones puede contener una columna de titulos, especialmente si se trata de una tabla de referencias cruzadas.
  • TFOOT: Pie de tabla, se utiliza para contener informacion complementaria a los datos del cuerpo, como totales, maximos, minimos, prodemios,etc...

多Como combinar celdas?

Las tablas normalmente poseen una estructura uniforme, pero en ciertas ocasiones es necesario combinar mas de una celda, expandiendola mas de una columna o mas de una fila, para esos casos debemos utilizar los atributos colspan="..." y rowspan="..." indicando como valor la cantidad total de celdas a ocupar. Cuando realizamos estos cambios el contenido correlativo se desplazara a la derecha.

Contenido Multimedia

Originalmente era poco comun ver este tipo de elementos dentro de una pagina, debido al ancho de banda manejado a単os anteriores. Otro de los motivos por lo que ocurria esto era la falta de compatibilidad con los navegadores y que en muchas ocasiones era necesario instalar algun plugin o complemento para que estos funcionen como por ejemplo: flash player.

多Multimedia sin plugins?

Los navegadores incorporan una API para este tipo de contenidos, solo debemos utilizar las etiquetas <audio> y <video>, que son compatibles con los formatos .mp3, .wav, .ogg y .mp4, .webm, .ogv respectivamente. Cabe destacar que debemos especificar el atributo controls si queremos que aparezcan los botones del reproductor, e incluso podemos utilizar otros atributos como autoplay muted loop

多Contenido Embebido?

Categoria web que se utiliza para identificar a todo contenido que haya sido compartido desde un sitio externo, con autorizacion del autor original; o simplemente a aquel contenido que no pertenece al universo HTML y se esta utilizando dentro de la pagina. Para la mayoria de los casos se utiliza la etiqueta <iframe> que permite la carga asincrona de contenido.
