Formularios HTML

Son herramientas que permiten el intercambio de datos entre cliente y servidor y para poder utilizarlos dentro de una pagina web debemos colocar la etiqueta <form> y dentro de la misma los elementos que van a ser procesados para el envio, cuando el usuario ejecute dicha accion.

¿Como funcionan?

A la etiqueta mencionada anteriormente debemos colocarle por lo menos 2 atributos, entre ellos se encuentran action="..." que indica lo que debe suceder tras realizar el envio del formulario, normalmente se utiliza un script en lenguaje de servidor; y method="..." que indica como se va a realizar el mismo, entre los metodos mas comunes se encuentran:

  • GET: Metodo poco seguro cuya prioridad radica en la respuesta del servidor, los datos pernacen visibles en la barra de direcciones y se utiliza principalmente en Buscadores
  • POST: Metodo seguro ya que se encriptan los datos antes de realizar el envio y son visibles dentro de la etiqueta, se utiliza en formularios de Resgistro, Inicio de sesion, Contacto, etc...

¿Como estan compuestos?

Dentro del formulario podemos utilizar cualquiera de las etiquetas conocidas, pero entre las utilizadas destacan las Etiquetas (LABEL) que indican que debe escribir/seleccionar el usuario y facilitan la accesibilidad; y por otro lado estan los Campos (INPUT / TEXTAREA / SELECT) que corresponden a los elementos donde el usuario puede ingresar/marcar las opciones indicadas por los label, entre los atributos mas comunes de los campos se encuentran los siguientes:

  • name="..." valor que identifica al campo de formulario, permite la recoleccion de los datos ingresados y determinar el grupo de pertenencia de los campos.
  • type="..." atributo que debe figurar en los campos de tipo input e indica cual es el tipo de dato valido, en caso de las otras etiquetas viene implicito.
  • value="..." atributo que indica el valor del campo, este es implicito cuando el usuario completa con sus datos, pero debe figurar cuando selecciona algun valor.

Contenido Multimedia

Nos referimos con este nombre a todo contenido grafico y audiovisual que podemos incroporar en nuestras paginas web, ya sea que se encuentren en nuestro sitio u obtengamos dichos recursos desde otra direccion. Si bien los navegadores actuales son compatibles con dichos elementos, antiguamente se requeria de la instalacion de algun plugin o complemento, como por ejemplo flash player.

¿Contenido Embebido?

Categoria web que se utiliza para identificar a todo elemento que se incropora desde un sitio externo y que fue compartido con autorizacion del autor original. Para incorporar este tipo de elementos se suele utilizar la etiqueta <iframe> que permite la carga asincrona de contenido, es decir que no afecta los tiempos de carga de nuestra pagina.

¿Multimedia sin Plugins?

Como se menciona al principio, actualmente podemos incorporar estos elementos sin necesidad de instalar algun complemento, ya que solamente colocar la etiqueta <audio> o <video> e indicar un recurso compatible con los formatos .mp3, .wav, .ogg y .mp4, .webm, .ogv respectivamente. Debemos tener en cuenta que para que el usuario pueda reproducir este material debemos colocar el atributo controls.

Tablas HTML

Son herramientas que permiten organizar la informacion de manera tabular, deben utilizar cuando los datos se corresponden a una estructura determinada. Para incorporarlas solamente debemos utilizar la etiqueta <table> y dentro de las misma los elementos que forman parte de su estructura, respetando el modelo de cajas.

¿Poseen estructura semantica?

Dentro de la etiqueta mencionada anteriormente, debemos colocar las filas con la etiqueta <tr> (Table Row) y dentro de las mismas las celdas de contenido o encabezado utilizando <th>(Table Heading) y <td> (Table Division). Pero a partir de la version 5, se incorporo la estructura semantica, que se compone de los siguientes elementos:

  • THEAD: Cabecera de tabla que contiene la fila de encabezados.
  • TBODY: Cuerpo de tabla que contiene los datos y puede poseer encabezados.
  • TFOOT: Pie de tabla que contiene datos complementarios al cuerpo.
