Formularios HTML

Son elementos que permiten el intercambio de datos entre cliente y servidor, para utilizarlos en nuestro sitio debemos colocar la etiqueta <form> y dentro de la misma los elementos que seran procesados cuando se realice el envio del mismo.

¿Como Funcionan?

A la etiqueta mencionada anteriormente debemos colocarle por los menos 2 atributos, action="..." que indica lo que debe suceder cuando se realiza el envio del formulario; y method="..." que indica como se deben procesar los datos, es decir, quien tiene prioridad durante el envio de los mismos.

  • GET metodo que prioriza la respuesta del servidor, los datos permanecen visibles en la barra de direcciones y el envio se hace en texto plano. Se utiliza principalmente en Buscadores.
  • POST metodo que prioriza la seguridad de los datos, ya que cifra los mismos y estos no se visualizan en la barra de direcciones. Se utiliza en formularios de registro, acceso, contacto, etc...

¿Como estan Compuestos?

Dentro de los formularios podemos utilizar cualquier etiqueta conocida, pero exitens 2 tipos de etiquetas necesarias para el envio del mismo, las Etiquetas (LABEL) que indican al usuario lo que debe ingresar o seleccionar y tambien facilitan la accesibilidad; y los Campos (INPUT / TEXTAREA / SELECT) que corresponden a los controles del formulario, aquellos con lo que el usuario interactua y que poseen algunos de los siguientes atributos.

  • name="..." corresponde al nombre con el que se identifica el campo, es necesario para la extraccion de los datos. En caso de opciones multiples el nombre es compartido.
  • type="..." indica el tipo de valor que se puede ingresar o seleccionar, limita el campo para su completado. Es obligatorio cuando utilizamos la etiqueta <input>
  • value="..." hace referencia al dato ingresado, por defecto es implicito, pero si el usuario selecciona una opcion debe figurar dentro del campo de formulario.

Tablas HTML

Son elementos que permiten organizar la informacion de manera tabular y que se utilizan cuando los datos deben presentarse de una manera determinada. Para incorporarlas dentro de la pagina, simplemente, debemos colocar la etiqueta <table> y de las mismas colocaremos las filas y las columnas respetando el modelo de cajas.

¿Poseen Carga Semantica?

Dentro de la etiqueta mencionada anteriormente debemos colocar las etiquetas <tr> (Table Row) para definir las filas contenedoras y dentro de las mismas, las etiquetas <td> (Table Division) que representa las celdas de contenido. A partir de HTML5 se incorporo un conjunto de etiquetas que corresponde a una estructura semantica, que facilita a los motores de busqueda la identificacion del contenido, ademas que amplia la forma en la que podemos diseñar las tablas.

  • THEAD corresponde a la cabera de tabla y contiene la fila de encabezados, que se representan con las etiquetas <th> (Table Heading)
  • TBODY corresponde al cuerpo de tabla donde se encuentran los datos a representar y normalmente suele estar compuesto por varias filas.
  • TFOOT corresponde al pie de tabla y posee datos complementarios al conteido, como totales, promedios, maximos, minimos, etc...

¿Celdas combinadas?

En ciertas ocasiones podemos necesitar que una celda ocupe mas de una columna o mas de una fila, para ello debemos utilizar los atributos colspan="..." y rowspan="..." respectivamente, indicando la cantidad total de celdas a ocupar. Contenido Multimedia

Se conoce con este nombre a todo elemento audiovisual que podemos incorporar dentro de nuestra pagina web. Los navegadores actuales soportan de manera nativa este tipo de elementos y ya no es necesaria la instalacion de plugins o complementos de terceros, como por ejemplo flash player. Si queremos corroborar que este u otro tipo de elementos que incorporamos a nuestro sitio, es compatible con el navegador podemos hacerlo desde el sitio Can I Use

¿Contenido Embebido?

Este nombre corresponde a una categoria web que indica que el elemento utilizado fue compartido con autorizacion del publicador original. Para ello, normalmente se utiliza la etiqueta <iframe que permite la carga asincrona de contenido, es decir, que no afecta a los tiempos de carga de nuestro sitio y permite mostrar, dentro de una ventana, contenido externo. Podemos reconocer este tipo de elementos gracias a las palabras compartir, embeber, codigo de insercion, incorporar, incrustar, etc...

¿Multimedia sin Plugins?

Como se indica mas arriba, ya no es necesario instalar complementos en el navegador para este tipo de contenidos, ya que los navegadores actuales incluyen una API de reproduccion y para utilizarla simplemente debemos colocar las etiquetas <audio y <video> que son compatibles con los formatos .mp3, .wav, .ogg y .mp4, .webm, .ogv respectivamente. Estas etiquetas tambien soportan atributos como autoplay muted loop y controls

Acceso FTP

Se conoce con este nombre al Protocolo de Transferencia de Archivos utilizado para descargar o subir contenido al servidor de hosting. Cuando contratamos dicho servicio, nuestro proveedor nos brindara la informacion necesaria para intercambiar datos entre nuestra computadora y el servidor, mencionada a continuacion, por correo electronico.

  • HOST: Nombre o direccion IP del servidor al cual vamos a enviar nuestros archivos.
  • USERNAME: Nombre con el que se identifica a la persona autorizada a transferir datos.
  • PASSWORD: Contraseña que valida la identidad del usuario que establece la conexion.
  • PORT: Canal de comunicaciones que identifica como debe comunicarse con el servidor web.
