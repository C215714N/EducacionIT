Formularios HTML

Son herramientas que permiten el intercambio de datos entre cliente y servidor, para utilizarlos dentro de la pagina simplemente debemos colocar la etiqueta <form> y dentro de la misma los elementos que seran procesados para el envio de los datos.

¿Como funcionan?

A la etiqueta mencionada anteriormente debemos especificarle por lo menos 2 atributos, que son action="..." el cual indica que debe suceder al enviar el formulario; y method="..." que indica como se debe realizar el intercambio de los datos.

  • GET: metodo cuya prioridad radica en la respuesta del servidor, ya que los datos se envian en texto plano y permanecen visibles en la barra de direcciones. Se utiliza principalmente en Buscadores
  • POST: metodo cuya prioridad esta en proteger los datos enviados por el usuario, ya que cifra los mismos antes de realizar el envio. Se utiliza en formularios de registro, acceso, contacto, etc...

¿Como estan compuestos?

Dentro del formulario podemos colocar cualquier etiqueta, pero necesariamente vamos a utilizar Etiquetas (LABEL) que funcionan a modo de rotulo, para que el usuario sepa como debe completar el mismo; y los Campos (INPUT / TEXTAREA / SELECT) que permiten al asuario seleccionar o completar con sus datos.

  • name="..." corresponde al nombre con el que identificara el campo, este puede ser compartido en caso de selecciones de opcion.
  • type="..." corresponde al tipo de dato admitido por el campo de formulario, en caso de los input es obligatorio.
  • value="..." corresponde al valor que sera enviado al utilizar el formulario y es obligatorio en caso que el usuario seleccione.

Contenido Multimedia

Se identifica con este nombre a todo elemento audiovisual que podemos utilizar dentro de una pagina web. Antiguamente para poder implementar este tipo de elemtos era necesaria la instalacion de algun plugin o complemento de terceros, debido a que los navegadores no brindaban soporte nativo. Pero con la ultima actualizacion de HTML, los navegadores incluyen APIs de reproduccion de contenido.

¿Contenido Embebido?

Se conoce con este nombre a una categoria web que identifica a un elemento que fue compartido con autorizacion del publicador original. Normalmente podemos incrustar contenido gracias a la etiqueta <iframe> que permite la carga asincrona de contenido.

¿Multimedia sin Plugins?

Como se menciona anteriormente, hoy en dia, los navegadores permiten agregar este tipo de contenido gracias a las etiquetas <audio> y <video> que soportan los formatos .mp3, .wav, .ogg y .mp4, .webm, .ogv respectivamente. Ademas podemos especificar atributos como controls, autoplay, muted, loop, etc... segun sea necesario.

Tablas HTML

Son elementos que permiten mostrar informacion de manera tabular, para utilizarlas simplemente debemos colocar la etiqueta <table> y dentro de la misma los elementos que la conforman respetando el modelo de cajas.

¿Estructura Semantica?

Dentro del contenedor, mencionado anteriormente, debemos colocar las filas (Table Row) y dentro de las mismas las celdas (Table Division), donde colocaremos la informacion. A partir de la ultima version de HTML, se incorporaron elementos semanticos que facilitan la interpretacion de las partes de la tabla.

  • THEAD: Cabecera de tabla, contiene los encabezados de la misma. Estos se representan con la etiqueta <th> (Table Heading).
  • TBODY: Cuerpo de tabla, contiene los datos a representar, tambien puede contener encabezados si se trata de una tabla de referencias cruzadas.
  • TFOOT: Pie de Tabla, contenido complementario al cuerpo. Puede contener totales, promedios, maximos, minimos, etc. A diferencia del resto, no es obligatorio.
