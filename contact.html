Formularios HTML

Son herramientas que permiten la interaccion entre cliente y servidor (intercambio de datos), para poder implementarlos debemos utilizar la etiqueta <form> y dentro de la misma colocar los elementos que van a formar parte del envio, sobre los cuales el usuario debera seleccionar o ingresar los datos pertinentes.

¿Como funcionan?

A la etiqueta mencionada anteriormente debemos agregarle por lo menos 2 atributos, por un lado action="..." que indica al navegador que se debe realizar cuando el usuario envia el formulario, por lo general se utiliza un script en lenguaje de backEnd (ej: php); por otro lado method="..." indica cual es la prioridad y como debe realizarse el envio.

  • GET: Metodo menos seguro, que da prioridad a la respuesta del servidor, la informacion enviada puede visualizarse desde la barra de direcciones y es el metodo mas rapido.
    • Buscador Consulta Query
  • POST: Metodo mas seguro debido a que encripta los datos antes de realizar el envio y cuya prioridad es resguardar la informacion del usuario, por este motivo es mas lento.
    • Registro Inscripcion
    • Acceso Inicio Sesion
    • Contacto Consulta Web

¿Cuales son sus elementos?

Dentro de un formularios encontramos 2 elementos, por un lado estan las etiquetas(LABEL) o textos que funcionan como rotulos que indican al usuario lo que debe seleccionar o escribir, tambien facilitan la accesibilidad; por el otro estan los campos(INPUT / TEXTAREA / SELECT) que permiten al usuario escribir o seleccionar datos y algunos de sus atributos mas caracteristicos son type="..." que indica cual es el tipo de dato a utilizarse y value="..." que se refiere al dato explicito.

Tablas HTML

Son elementos que permiten organizar la informacion de manera tabular, se componen de filas, columnas y celdas. Deben ser utilizadas cuando necesitamos exponer datos respetando cierta estructura o formato, indistintamente del tamaño de pantalla o espacio restante disponible, utilizando la etiqueta <table>.

¿Cual es su estructura?

Podemos encontrar 2 tipos de elementos, los identificadores que se utilizan para armar la tabla como <tr> (Table Row) y <th> (Table Division) y por otro lado estan los elementos semanticos como <THEAD>, <TBODY> y <TFOOT> que facilitan a los motores de busqueda el analisis del contenido de las tablas.

Contenido Multimedia

En la actualidad podemos incorporar estos elementos de forma nativa, gracias a que los navegadores incluyen una API (Application Programming Interface) de reproduccion. Tambien podemos agregar elementos externos, igual a como se hacia antes, pero en este ultimo caso el mismo debe pertenecer a una categoria conocida como embed (embebido) que indica a los motores de busqueda que dicho material fue compartido con autorizacion del sitio involucrado.

¿Contenido Embebido?

Como se menciona anteriormente, esta categoria corresponde a todo contenido ajeno a nuestro sitio que fue compartido desde la web del autor. Normalmente se utiliza una opcion como compartir, codigo de insercion, etc... que nos copia al portapapeles una etiqueta <iframe> con un link a la web especificada, dentro de este link encontraremos la palabra embed.
