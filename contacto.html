Formularios HTML

Son herramientas que permiten el intercambio de datos entre cliente y servidor, para implementarlos debemos utilizar la etiqueta <form> y solamente se tendra en cuenta para el envio, los datos que se encuentren dentro de la misma. Cabe destacar que es necesario especifar algunos atributos para definir como se deben manipular los datos y que se debe hacer con los mismos.

¿Como funcionan?

La etiqueta mencionada anteriormente posee 2 atributos necesarios para la manipulacion de datos, por un lado se encuentra action="..." que indica al navegador cual es la operacion a realizar cuando el usuario presiona el boton enviar, en la actualidad se utilizan scripts en lenguaje de servidor (ej: php) para que este ultimo resuelva que hacer; por otro lado esta method="..." que recibe al menos 2 valores segun la prioridad que deba darse a los datos y su respuesta respectivamente.

  • GET: Metodo que se utiliza para obtener informacion del servidor, la prioridad radica en la respuesta de este ultimo y debido a esto es bastante rapido, pero menos seguro debido a que los datos se envian en texto plano y siempre estan visibles en la barra de direcciones.
    • Buscadores - Consulta Query
  • POST: Metodo que se utiliza para enviar informacion al servidor de manera segura, ya que la prioridad radica en el cliente. Se encriptan los datos para la comunicacion, pero es mas lento debido a esto.
    • Inscripcion - Registro
    • Acceso - Inicio Sesion
    • Consulta Web - Contacto

Tablas HTML

Son herramientas que permiten organizar la informacion de manera tabular y se deben implementar cuando los datos que se representan en pantalla requieren de cierta estructura. Para utilizarlas simplemente colocamos la etiqueta <table>, pero a diferencia de otros ambitos donde se utilizan tablas, en html se deben crear siguiendo el modelo de cajas utilizando etiquetas que funcionan como contenedores y otras que funcionan como elementos de contenido.

¿Cual es su estructura?

A la etiqueta mencionada anteriormente se la divide en 3 partes, la cabecera (THEAD) donde colocamos los titulos de la tabla, el cuerpo (TBODY) donde colocamos los datos de interes para el usuario, y el pie (TFOOT) que es opcional y donde colocamos datos complementarios a la tabla, como totales, estadisticas, etc. A la vez, debemos indicar cuando empieza una fila con la etiqueta <tr> (TABLE ROW) y dentro de cada fila cuantas celdas va a contener con las etiquetas <td>(TABLE DIVISION) <th>(TABLE HEADING) para contenido y titulos respectivamente.

Contenido Multimedia

¿Contenido Embebido?

Categoria web (embed) que se utiliza para indicar que dicho elemento fue compartido desde otro sitio, el mismo puede pertenecer o no al universo HTML. Normalmente se puede compartir este contenido utilizando la etiqueta <iframe> (Marco flotante) que se utiliza para insertar contenido en nuestra web de manera asincrona, es decir, que no afecta a los tiempos de carga de nuestra pagina.

¿Multimedia sin plugins?

Actualmente los navegadores incorporan una API de reproduccion a las que podemos acceder gracias a las etiquetas <audio> y <video> que soportan los formatos .mp3 - .wav - .ogg y .mp4 - .webm - .ogv respectivamente. Para poder visualizar el reproductor en pantalla debemos colocar el atributo controls y en el caso de los videos podemos utilizar autoplay muted loop si queremos se reproduzcan de manera indefinida.
