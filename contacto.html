Formularios HTML

Son elementos que permiten una interaccion real (intercambio de datos) entre cliente y servidor, a diferencia de otras etiquetas que permiten al usuario interactuar con el sitio, estos no devuelven los mismos resultados ya que dependen de los datos ingresados por el usuario para generar una respuesta. Para poder implementar un formulario en nuestra pagina, debemos utilizar la etiqueta <form> teniendo en cuenta que solamente se enviaran a traves del mismo los datos que se encuentra dentro del elemento.

¿Como Funcionan?

La etiqueta mencionada anteriormente debe contar con al menos 2 atributos principales, estos sonaction="..." y method="...". En caso del primero, indica al navegador que debe suceder con los datos ingresados, normalmente se utiliza un script en lenguaje de servidor como por ejemplo PHP y es este ultimo quien se encarga de operar con los mismos ademas de elaborar una respuesta. Con respecto al otro atributo, indica como se deben procesar los datos y la prioridad de los mismos, existen 2 valores posibles que podemos utilizar a la hora de enviar nuestro formulario.
El primer valor es method="GET" cuya prioridad radica en obtener la respuesta del servidor, por lo tanto los datos del usuario no son relevantes y por eso no los encripta, admite hasta 8kb y la informacion es visible desde la barra de direcciones. El otro valor el method="POST" cuya prioridad radica en proteger lo maximo posible los datos del cliente, por eso encripta los datos antes de realizar el envio y admite hast 8mb.

¿Como estan compuestos?

Dentro de los formularios encontramos 2 tipos de elementos, Las ETIQUETAS (label) que funcionan como rotulos indicadores de lo que debe completar o seleccionar el usuario, tambien permiten a los navegadores el indexado del formulario, asi como tambien la facilidad de lectura para los screenreader; y Los CAMPOS (input, textarea, select) que permiten al usuario interactuar con el sitio para el ingreso o seleccion de datos, es importante entender que el atributo value="..." es lo que se considera cuando se realiza el envio y en caso de que el usuario no escribiera dentro el mismo quedara vacio. Tambien debemos indicar el tipo de dato admitido para reforzar la validacion del formulario, ademas de impedir el envio de formularios vacios.
