Formularios HTML

¿Para que sirven?

Son herramientas que se utilizan para el intercambio de datos entre un cliente y un servidor (interaccion real), existen diferentes tipos de formularios, y se pueden clasificar segun su objetivo. Estan aquellos cuya funcion es obtener datos de un servidor y luego estan aquellos cuya mision es enviar y guardar datos a un servidor

  • Buscadores - Consulta Query(GET)
  • Inscripcion - Registro(POST)
  • Acceso - Inicio Sesion(POST)
  • Contacto - Consulta Web(POST)

¿Como funcionan?

Para utilizarlos, es necesario colocar el contenido que queremos enviar/recibir dentro de una etiqueta <FORM>, en la misma debemos especificar 2 atributos action="..." y method="..."que indican que debe hacerse con la informacion al enviarse, y como se debe llevar a cambo el proceso.

  • ACTION: Antiguamente se utilizaba el valor mailto:usuario@servidor, pero debido a que necesitamos que el cliente cuente con una aplicacion de correo, hoy en se utiliza algun script en lenguaje de servidor como PHP.
  • METHOD: Existen 2 valores posibles para este atributo, por un lado GET prioriza la respuesta del servidor, es menos seguro/rapido y la informacion enviada es visible desde la barra de direcciones. POST por el contrario, es mas seguro/lento debido a que encripta los datos antes de enviarlos, prioriza el envio del cliente y la respuesta puede no ser inmediata.

¿Cuales son sus componentes?

Los formularios se componen de 2 tipos de elementos, por un lado las etiquetas(LABEL) que funcionan como rotulos que indican que datos debemos ingresar o seleccionar; y por el otro estan los campos(INPUT | TEXTAREA | SELECT) que funcionan como areas en las que el usuario puede ingresar o seleccionar datos, cabe destacar que a cada elemento le corresponde un tipo de dato.

  • Campo de Texto (Aa123!#)
  • Campo Numerico (123456)
  • Campo de e-mail (*@*)
  • Campo Contraseña(****)
  • Campo fecha(dd/mm/aaaa)
  • Campo opción Única si | no
  • Campo opción Multiple Arte | Música | Deportes
  • Campo Multilinea
  • Lista Desplegable

Tablas HTML

¿Cuando utilizarlas?

Las tablas son elementos que permiten organizar la informacion de manera tabular y deben ser utilizadas cuando necesitamos que los datos representados en pantalla siempre respeten determinada estructura. Es importante tener en cuenta que NO debemos utilizar las tablas para diseñar la pagina como se solia hacer antes de la llegada de CSS.

¿Por qué utilizar tablas semanticas?

Con el surgimiento de HTML5, la w3c incorporo nuevas etiquetas para utilizar dentro de las tablas. Estas funcionan como separadores de contenido que facilitan a los motores de busqueda identificar las partes de las mismas y reemplazan en la mayoria de los casos a los clasicos <tr>.

  • tHEAD: Representa la cabecera de tabla y se utiliza para contener los titulos, a su vez en lugar de utilizar la etiqueta <td> debemos implementar <th> para los mismos.
  • tBODY: Representa el cuerpo y donde se coloca la informacion a representar, dependiendo del tipo de tablas, podemos implementar titulos si trabajamos con una tabla de referencias cruzadas; en esta seccion de la tabla todavia se siguen implementando las etiquetas <tr> y <td>.
  • tFOOT: Representa al pie de tabla, el cual contiene la informacion complementaria al tBody. Es opcional y se utiliza para representar totales, promedio, o cualquier valor que haga referencia al contenido, tambien puede poseer celdas de titulo.

Contenido Multimedia

¿Que es el contenido Embebido?

Existe una etiqueta que se suele utilizar mucho cuando necesitamos incorporar a nuestra pagina web un mapa, un video, una presentacion o cualquier archivo externo que se pueda incorporar dentro de HTML, y es iframe (marco flotante) que funciona de manera asincronica, es decir, que los tiempos de carga de la misma NO afectan a nuestra pagina y es una manera de compartir contenido mediante la web.

¿Existen los Sonidos en la web?

Uno de los elementos incorporados en HTML5 es la etiqueta <AUDIO> que permite la reproduccion nativa de este tipo de contenido, sin necesidad de la descarga de algun plugin o reproductor externo. Los formatos admitidos por la API de reproduccion son .mp3 - .wav - .ogg, aunque tambien admite formatos de video como los que mencionan mas adelante, debido a que este elemento, solamente toma la pista de audio.

¿Videos sin necesidad de Plugins?

Como se menciono anteriormente, hoy en dia es posible reproducir contenido multimedia, sin utilizar herramientas de terceros, tal y como sucedia antes con YouTube y el discontinuado FlashPlayer. Los formatos admitidos son .mp4 - .webm - . ogv, asi como tambien, los formatos de audio. En este ultimo caso, el reproductor cambiara a su version de audio.
