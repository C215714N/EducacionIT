Formularios HTML

Son herramientas que permiten una interaccion real (intercambio de datos) entre cliente y servidor , ya que a diferencia de otros elementos como los hipervinculos , los formularios generan una respuesta diferente en base a los parametros definidos por el desarrollador y los valores completados por el usuario. Para poder implementar un formulario en nuestra pagina, es necesario utilizar la etiqueta <form>

¿Como Funcionan?

Dentro de la etiqueta mencionada anteriormente es necesario utilizar, por los menos, 2 atributos. Por un lado tenemos action = "..." que se utiliza para definir que sucedera con la informacion completada en el mismo formulario al presionar el boton enviar ; antiguamente se utilizaba un valor action="mailto:usuario@dominio" para que el cliente utilizara su aplicacion de correo predeterminada, pero hoy en dia se utiliza algun script en lenguaje de servidor (PHP).

Por otro lado, tenemos el atributo method="" que en definitiva define cual es la prioridad en el envio y como se va a trabajar la informacion; existen 2 valores posibles y estos son method="GET" , un metodo menos seguro, cuya informacion es accesible desde la barra de direcciones y admite hasta 8kb, y method="POST" el cual se considera seguro debido a que encripta los datos del usuario antes de enviar el formulario, admite hasta 8mb y la informacion no es accesible desde la barra de direcciones.

¿Como estan compuestos?

Dentro de un formulario encontraremos 2 elementos principales, por un lado tenemos las Etiquetas (LABEL) cuya finalidad es indicar al usuario que informacion debe colocar dentro del formulario, asi como tambien que opciones debe elegir; tambien se utiliza para los lectores de pantalla y SEO. Por otro lado, existen los Campos (INPUT, TEXTAREA, SELECT) que son los elementos donde el usuario puede escribir o seleccionar opciones, es importante destacar que existen diferentes tipos, segun los valores que permitamos ingresar al usuario, destacando principalmente los campos de tipo input.

Tablas HTML

Son elementos que permiten organizar la informacion de manera tabular y a diferencia de la mayoria de los programas, en el Lenguaje HTML estas se crean a partir del modelo de cajas , es decir que en lugar de estructurar filas y columnas , contemplaremos contenedores y contenido. Antiguamente, antes que se definiera CSS como lenguaje de diseño, era muy comun que las paginas web se crearan en base a trablas para la organizacion del contenido; hoy en dia se utilizan para presentar datos y al igual que la mayoria de las etiquetas HTML no poseen bordes visibles.

¿Estructura clasica?

Como se menciona anteriormente, la estructura de las tablas depende del modelo basado en jerarquias, en el cual para crear una tabla debemos utilizar un contenedor principal que es la etiqueta <TABLE> y dentro de la misma utilizaremos las etiquetas TR (Table Row) para definir cada fila individualmente y TD (Table Division) para indicar cada celda dentro de cada fila. Cabe destacar que a diferencia del resto de las etiquetas de bloque que se adaptan a su contenedor, las tablas se adaptan a su contenido

¿Estructura Semantica?

Con el surgimiento de HTML5 se incorporaron nuevas etiquetas a la estructura de las paginas web, entre las etiquetas involucradas se encuentran las tablas. Asi como podemos segmentar el body en header - main - footer, una tabla se puede segmentar en las siguientes etiquetas:

  • THEAD: Es la cabecera de la tabla, representa la fila de titulos y debe tener en su interior celdas de titulos que se identifican con la etiqueta TH (Table Heading). Estas etiquetas son obligatorias para definir el contenido.
  • TBODY: representa el cuerpo de la tabla, en esta etiqueta se sigue utilizando la estructura clasica, aunque en ocasiones se pueden utilizar celdas de titulo en su interior, especialmente si se esta representando una tabla de referencias cruzadas . Es donde se colocan los datos pertinentes para el usuario.
  • TFOOT: el pie de tabla, es opcional, y representa una fila complementaria a los datos presentado anteriormente. puede contener algun titulo y normalmente se utiliza para los totales, porcentajes o promedios

Contenido Multimedia

Desde que surge la web como tal, se pueden compartir elementos multimedia como audio y video , pero no siempre se podia visualizar el mismo dentro de los sitios que los contenian. Muchas veces era necesario el uso de plugins o complementos que debian instalarse en el navegador para reproducitr este tipo de contenido, uno de los mas populares en su tiempo fue Flash.

¿Contenido Embebido?

Una de las primeras etiquetas que permitia compartir contenido desde otra web, e incluso variar su contenido, fue iFrame (Marco Flotante) que mostraba una ventana dentro de nuestra pagina, en la cual se cargaba el contenido de forma asincronica. Actualmente esta etiqueta se usa para lo que se conoce como Contenido embebido, es decir elementos que no pertenecen al universo HTML, pero se pueden utilizar dentro de nuestra web y tambien hace referencia a los elementos que se comparten desde un sitio externo. Existen otras formas de insertar contenido multimedia como se mencionaran mas adelante.

¿Sonidos en la Web?

Si bien, es cierto que desde hace mucho que se pueden utilizar pistas de audio en un sitio web como se menciono anteriormente, es necesario entender que con la incorporacion del nuevo standard se agrego a los navegadores una API de reproduccion de contenido multimedia. Para utilizar sonidos, solamente debemos implementar la etiqueta AUDIO, la cual admite los formatos .mp3 - .wav - .ogg , aunque dependiendo del navegador puede llegar a soportar otros formatos. Una de las caracteristicas fundamentales de este elementos es que tambien reproduce formatos de video, donde solamente se utiliza la pista de audio del mismo.

¿Multimedia sin plugins?

A diferencia del Elemento audio el elemento video no requiere del atributo controls para su visualizacion, pero al no poseer los mismos el video no se podra reproducir a menos que indiquemos el valor autoplay muted . Entre los formatos soportados se encuentran .mp4 - .webm - .ogv aunque tambien podemos cargar formatos de audio, pero el reproductor cambiara al mostrado en la etiqueta anterior.
