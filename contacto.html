Formularios HTML

Son herramientas que permiten el intercambio de datos entre cliente y servidor. La etiqueta que permite su implementacion es <form> y solamente se tendra en cuenta para el envio de datos lo que se encuentre dentro de la misma. Cabe destacar que es necesario especifar algunos atributos para definir como se deben manipular los datos y que se debe hacer con los mismos.

¿Como funcionan?

La etiqueta mencionada anteriormente posee 2 atributos necesarios para la manipulacion de datos, por un lado se encuentra action="..." que indica al navegador cual es la operacion a realizar cuando el usuario presiona el boton enviar, en la actualidad se utilizan scripts en lenguaje de servidor (ej: php) para que este ultimo resuelva que hacer; por otro lado esta method="..." que recibe al menos 2 valores segun la prioridad que deba darse a los datos y su respuesta respectivamente.

  • GET: se utiliza cuando la prioridad radica en la respuesta del servidor, es menos seguro debido a que los datos se visualizan el la barra de direcciones, pero es mucho mas rapido debido a que los datos se envian en texto plano y debido a esto su uso es mas acotado.
    • Buscadores - consulta Query
  • POST: se utiliza cuando la prioridad radica en los datos propiciados por el cliente, es mas seguro debido a que encripta los datos antes de realizar el envio y soporta el envio de archivos, pero es mas lento. Este metodo es el preferido para el envio para los diferentes tipos de formularios.
    • Inscripcion - Registro
    • Acceso - Inicio sesion
    • Consulta Web - Contacto

¿Como estan compuestos?

Dentro de los formularios podemos encontrar 2 tipos de elementos, por un lado tenemos las Etiquetas (LABEL) que indican al usuario que dato debe seleccionar/ingresar y facilitan la accesibilidad; y por otro estan los campos (INPUT/TEXTAREA/SELECT) donde el usuario debe realizar las acciones propiamente dichas. Los campos poseen 2 atributos que, aunque no siempre estan presentes, son type="..." indicando el tipo de dato que se debe ingresar (ej: texto/numero) y value="..." que representa el valor del control, haya sido ingresado por el usuario o este de manera predeterminada.

Tablas HTML

Son herramientas que se utilizan para organizar la informacion de manera tabular y deben ser implementadas cuando los datos se corresponden con determinada estructura. Para crear las mismas se utiliza la etiqueta <table> respetando el modelo de cajas, es decir, utilizando etiquetas como contenedores y otras como contenido.

¿Cual es su estructura?

La etiqueta mencionada anteriormente contiene 3 partes fundamentales, la cabecera de tabla <thead>, el cuerpo de tabla <tbody> y el pie de tabla <tfoot>(opcional). Cada una de estas partes estan divididas en filas <tr> (Table Row) y celdas <td> (Table Division), pero a diferencia de cualquier otra representacion de tablas, las columnas, si bien existen, no son representadas mediante una etiqueta.

Contenido Multimedia

Desde que existen los sitios web y los medios audiovisuales se ha intentado incorporar estos ultimos en las paginas propiamente mencionadas; y con el paso del tiempo se ha logrado hacer de manera tal que ya no sea necesario instalar ningun plugin o herramientas de terceros para que nuestro sitio pueda reproducir dicho material sin problemas de compatibilidad.

¿contenido embebido?

Se conoce como embed al contenido incorporado a nuestra pagina desde otro sitio o que directamente no pertenece al universo html. Es una categoria web para identificar el contenido que fue compartido desde otra web y suele implementarse utilizando la etiqueta <iframe> que permite la carga asincronica de contenido, para no afectar los tiempos de carga de nuestra pagina.

¿Multimedia sin Plugins?

En la actualidad los navegadores incorporan API's de reproduccion de contenido a las que podemos acceder gracias a las etiquetas <audio> y <video> soportando los formatos de audio .mp3 - .wav - .ogg y los formatos de video .mp4 - .webm - .ogv; Cabe destacar que los reproductores seran visibles si utilizamos el atributo controls y podemos reproducir el material de manera indefinida utilizando autoplay loop
