Formularios HTML

¿Cual es su función?

Son herramientas que permiten una interaccion real (intercambio de datos) entre el cliente y el servidor y permiten al usuario enviar una peticion directamente desde la pagina, que segun su tipo puede o no ser respondida inmediatamente. Para poder utilizarlos debemos utilizar la etiqueta <form> y solamente se procesara para esta peticion los datos que se encuentren dentro de la misma; tambien tenemos que especificar determinados atributos que se mencionan mas adelante , asi como tambien los elementos en su interior.

  • Buscador - Consulta Query
  • Inscripcion - Registro
  • Acceso - Inicio Sesion
  • Contacto - Consulta Web

¿Como funcionan?

A la etiqueta mencionada anteriormente debemos colocarle por los menos 2 atributos; por un lado tenemos el atributo action="..." que indica cual es la accion que debe realizarse al momento de enviar el formulario, actualmente se utiliza un script en lenguaje de servidor (ej: PHP) para garantizar el correcto procesamiento de la peticion. Por el otro, encontramos al atributo method="..." que indica como debe realizarse el envio de los datos mediante HTTP y cual es la prioridad de los mismos.

  • GET: Metodo menos seguro debido a que la informacion permanece visible en la barra de direcciones, pero mas rapido ya que prioriza la respuesta del servidor ante todo.
  • POST: Metodo mas seguro ya que encripta los datos enviados por el usuario, pero al realizar este proceso es mucho mas lento que el anterior.

¿Que elementos los componen?

Dentro de los mismos encontramos 2 elementos diferentes, por un lado estan las Etiquetas(LABEL) que funcionan a modo de rotulo, indicando al usuario que datos debe ingresar/seleccionar dentro del formulario, tambien facilita la accesibilidad. Por el otro lado estan los Campos (INPUT / TEXTAREA / SELECT) que representan las areas de la pagina donde el usuario puede ingresar/seleccionar los datos que van a ser enviados, estos poseen 2 atributos fundamentales que son type="..." que indica el tipo dato a ingresar y value="..." que representa el dato ingresado/seleccionado.

Tablas HTML

¿Para qué sirven?

Son herramientas que se utilizan para representar datos de manera tabular, deben implementarse siempre y cuando estos necesiten ser visualizados de una manera en concreto sin modificaciones. Las mismas se crean a partir del modelo de cajas, es decir, utilizando etiquetas que cumplen la funcion de contenedos y otras que cumplen la funcion de contenido. A diferencia de otras etiquetas de bloque, las tablas se adaptan al tamaño de su contenido.

¿Poseen estructura semantica?

Efectivamente, con la aparicion de HTML5 se incorporaron nuevos elementos que facilitan a los motores de busqueda el analisis e indexado del contenido de las tablas. En su mayoria estos nuevos elementos reemplazan a las filas <tr> y a las celdas <td>

  • THEAD Cabecera de tabla que se utiliza para la representacion de los titulos, en su interior deben colocarse celdas de titulo <TH>
  • TBODY Cuerpo de tabla, representa el contenido principal, es la unica parte que mantiene la estructura clasica y en ciertas ocasiones podemos utilizar celdas de titulo, por ejemplo: tablas de referencia cruzada
  • TFOOT Pie de tabla, parte opcional de las tablas que se utiliza a modo de detalle, ya sea para mostrar totales, promedios, etc. y por ende es complementaria al cuerpo.

Contenido Multimedia

¿Contenido Embebido?

Antiguamente si queriamos incorporar contenido multimedia dentro de nuestra pagina, debiamos contar con elementos externos (plugins, complementos) ya que los navegadores no contaban con APIs incorporadas para la reproduccion de este tipo de contenido. En caso de utilizar contenido externo podiamos implementar la etiqueta <iframe> que permite la insercion de contenido de manera asincrona.
