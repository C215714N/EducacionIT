Formularios HTML

Son elementos que permiten una interaccion real (intercambio de datos) entre cliente y servidor, a diferencia de otras etiquetas que permiten al usuario interactuar con el sitio, estos no devuelven los mismos resultados ya que dependen de los datos ingresados por el usuario para generar una respuesta. Para poder implementar un formulario en nuestra pagina, debemos utilizar la etiqueta <form> teniendo en cuenta que solamente se enviaran a traves del mismo los datos que se encuentra dentro del elemento.

¿Como Funcionan?

La etiqueta mencionada anteriormente debe contar con al menos 2 atributos principales, estos son action="..." y method="...". En caso del primero, indica al navegador que debe suceder con los datos ingresados, normalmente se utiliza un script en lenguaje de servidor como por ejemplo PHP y es este ultimo quien se encarga de operar con los mismos ademas de elaborar una respuesta. Con respecto al otro atributo, indica como se deben procesar los datos y la prioridad de los mismos, existen 2 valores posibles que podemos utilizar a la hora de enviar nuestro formulario.
El primer valor es method="GET" cuya prioridad radica en obtener la respuesta del servidor, por lo tanto los datos del usuario no son relevantes y por eso no los encripta, admite hasta 8kb y la informacion es visible desde la barra de direcciones. El otro valor el method="POST" cuya prioridad radica en proteger lo maximo posible los datos del cliente, por eso encripta los datos antes de realizar el envio y admite hast 8mb.

¿Como estan compuestos?

Dentro de los formularios encontramos 2 tipos de elementos, Las ETIQUETAS (label) que funcionan como rotulos indicadores de lo que debe completar o seleccionar el usuario, tambien permiten a los navegadores el indexado del formulario, asi como tambien la facilidad de lectura para los screenreader; y Los CAMPOS (input, textarea, select) que permiten al usuario interactuar con el sitio para el ingreso o seleccion de datos, es importante entender que el atributo value="..." es lo que se considera cuando se realiza el envio y en caso de que el usuario no escribiera dentro el mismo quedara vacio. Tambien debemos indicar el tipo de dato admitido para reforzar la validacion del formulario, ademas de impedir el envio de formularios vacios.

Tablas HTML

Son elementos que permiten organizar la informacion de manera tabular, al igual que en cualquier otro ambito, las mismas se componen de filas, columnas y celdas; pero a diferencia de la mayoria de los programas que utilizan tablas en informatica, HTML crea las tablas a partir del modelo de cajas, es decir que utiliza etiquetas que cumplen la funcion de contenedor y otras que cumplen la funcion de contenido. Para crear una tabla necesitamos implementar la etiqueta <table> y a diferencia del resto de las etiquetas de bloque, esta se adapta a su contenido en lugar de su contenedor.

Estructura Clasica

Como se menciona anteriormente, para crear una estructura de tablas es necesario respetar el modelo de cajas, es decir que dentro de la etiqueta principal utilizaremos por los 2 etiquetas mas. <tr> (Table Row) para disponer las filas que componen nuestra tabla y por otro lado <td> (Table Division) para colocar la informacion contenida en las celdas. Un detalle a destacar es que las tablas no poseen borde de manera predeterminada y tampoco poseen columnas como tales.

Estructura Semantica

Con la incorporacion del nuevo estandar, se agregaron nuevos elementos a las tablas para poder diferenciar por parte de los motores de busquedas el tipo de contenido de las mismas. Entre las etiquetas que se incorporaron encontramos <THEAD> que representa la cabecera de la tabla, es obligatoria y debe contener en su interior celdas de titulo representadas por la etiqueta <th>; <TBODY> que representa el cuerpo de la tabla, es la unica etiqueta que mantiene en su interior la estructura clasica representada anteriormente, aunque tambien puede contener celdas de titulo, especialmente si estamos armando una tabla de referencias cruzadas; y por ultimo, <TFOOT> que representa el pie de la tabla, contiene informacion complementaria a la misma, asi como algunos titulos y es opcional.

Contenido multimedia

Contenido Emebebido

Antiguamente para incorporar elementos externos en nuestra pagina web se solia utilizar la etiqueta <iframe> que permite la carga asincronica de contenido sea este perteneciente al mismo sitio o no. Hoy en dia existe una categoria web conocida como embed que engloba a todos los elementos compartidos que normalmente no pertecen al universo HTML.

Sonidos en la Web

Como se menciono anteriormente, para poder utilizar contenido multimedia en la web era necesario algun complemento. Hoy en dia podemos reproducir de forma nativa este tipo de elementos a traves de la API del navegador, los formatos soportados por la etiqueta <audio> son .mp3 - .wav - .ogg, aunque tambien admite formatos de video como los que se mencionan mas adelante. Para que se vea el reproductor necesitamos indicar el atributo controls.

Videos sin plugins

En la actualidad no es necesario instalar algun complemento o plugin para la reproduccion de este tipo de material, ya que los navegadores lo hacen de forma nativa. Esta, ademas de muchas otras, fue una de las razones por la cual el plugin de Flash Player fue quedando obsoleto hasta practicamente desaparecer de la web. Al igual que el elemento audio permite la reproduccion de sonidos, pero a diferencia del primero el aspecto del reproducto cambia. Los formatos admitidos son .mp4 - .webm - .ogv y en contraposicion al elemento iframe el contenido afecta los tiempos de carga.
