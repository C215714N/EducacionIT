Formularios HTML

¿Para que sirven?

Son una herramienta que permite la interaccion real, entre el cliente y el servidor, ya que a diferencia de otros elementos que tienen un comportamiento predeterminado, la informacion almacenada en estos dependera pura y exclusivamente del usuario. En la mayoria de los casos dependeremos de un lenguaje de backend para poder validar la informacion completada en los mismos, ya que los datos seran enviados a un servidor para que este determine que hacer con dicha informacion.

Dentro de un formulario vamos a encontrar diferentes elementos, que los podemos clasificar en Etiquetas (label) y Campos (select - textarea - input). El primero se utiliza para identificar que tipo de informacion debemos escribir o seleccionar del formulario y, por otro lado, el segundo es un conjunto de elementos de diferente tipo que nos permite completar con datos o seleccionar informacion previamente establecida.

  • Input TEXTO
  • Input NUMERO
  • input FECHA
  • input CORREO
  • input CONTRASEÑA
  • opcion UNICA
  • opcion MULTIPLE

¿Como funcionan?

Para poder enviar un formulario existen dos atributos fundamentales, que le indican al navegador que se debe hacer y de que manera, estos son action y method. Para saber que valor debemos utilizar, necesitamos tener en cuenta el tipo de formulario que vamos a implementar en nuestra pagina.

  • Buscador (Consulta Query)
  • Inscripcion (Registro)
  • Acceso (Inicio Sesion)
  • Consulta Web (Contacto)

Antiguamente, antes de la llegada de la banda ancha, era comun que los usuarios utilizaran aplicaciones de correo para descargar el mismo en las computadoras y por lo tanto los formularios antiguos utilizaban action="mailto:cristiandracedo@hotmail.com", pero debido a que hoy en dia esto es poco impractico, se utiliza un script en lenguaje de servidor. Por otro lado, exiten dos valores posibles para el method que se caracterizan por lo siguiente:

  • GET
    • Metodo menos seguro (No encripta los datos enviados)
    • Se implementa cuando la respuesta es lo mas importante y no tanto los datos
    • La informacion es accesible en todo momento
    • Admite hasta 8kb
  • POST
    • Metodo mas seguro (Cifra los datos al enviarse)
    • La informacion no es accesible directamente desde el navegador
    • Los datos enviados son la prioridad y no tanto la respuesta (que puede o no existir)
    • Admite hasta 8Mb
