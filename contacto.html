Formularios HTML

Son elementos que permiten una interaccion real (intercambio de datos) entre cliente y servidor, a diferencia de otras etiquetas como a (hipervinculos) cuya interaccion da como resultado un cambio de pagina (siempre el mismo archivo), los formularios permiten al cliente interactuar con el sitio, ya sea completando o seleccionando opciones para luego realizar una peticion al servidor que realizara una respuesta en base a los datos y configuracion definidos por el desarrollador.

Para implementar un formulario en una pagina es necesario utilizar la etiqueta <form> y definir diferentes parametros dentro de la misma, ademas de los elementos que compondran al mismo para que sea completado por los usuarios que visiten la pagina.

¿Como Funcionan?

Dentro de la etiqueta mencionada anteriormente se deben especificar por lo menos 2 atributos. Uno es action=".." que permite indicar al navegador que se debe hacer con la informacion del formulario cuando el usuario realiza el envio , antigualmente se utilizaba el valor: action="mailto:usuario@dominio" que permitia abrir la aplicacion de correos predeterminada del cliente para realizar el envio del mismo por correo; con la llegada de la banda ancha este metodo se volvio obsoleto porque dependemos que el cliente tenga configurada una aplicacion de correos.

Otro atributo es method=".." que se utiliza para indicar como debe realizarse el procesamiento de los datos y la importancia de los mismos, el mismo recibe 2 valores posibles. El primero es method="GET" que se utiliza cuando la prioridad esta en la respuesta otorgada por el servidor y los datos del usuario no son relevantes, admite hasta 8kb y los datos siempre estan accesibles desde la barra de direcciones. El otro valor el method="POST" y se utiliza cuando la prioridad esta en los datos otorgados por el cliente, admite hasta 8mb y encripta los datos antes de realizar el envio.

¿Como estan Compuestos?

Los formularios poseen 2 elementos principales, por un lado estan las Etiquetas (LABEL) que se utilizan para definir que tipo de datos debe ingresar o seleccionar el cliente dentro del formulario, tambien sirve para el SEO y los lectores de pantalla. Por otro lado, se encuentran los Campos (INPUT - TEXTAREA - SELECT) que permiten al usuario escribir o seleccionar opciones dentro del formulario, existen de diferentes tipos segun el dato que se deba ingresar y la etiqueta mas utilizada es el campo monolinea <INPUT />

Tablas HTML

Son elementos que permiten organizar la informacion de manera tabular y al igual que en cualquier ambito, las mismas se componen de filas y columnas , pero a diferencia del resto de los programas, en HTML se basan en el modelo de cajas para su creacion , es decir que debemos comtemplar que utiliza etiquetas de tipo contenedor y etiquetas de contenido . A diferencia del resto de las etiquetas de bloque , las tablas se ajustan al tamaño de su contenido y no al tamaño de su contendor.

¿Estructura Clasica?

Como se menciona anteriormente, las tablas en HTML son diferentes en su forma de creacion que en el resto de programas. Para definir una debemos crear un Contenedor <TABLE> que servira como base para crear nuestra tabla, luego por cada fila debemos insertar otro Contenedor de tipo <TR> (Table Row) y por ultimo dentro de cada fila Las etiquetas de Contenido <TD> (Table Division) por cada celda que posea la fila. Al igual que el resto de las etiquetas, las tablas no poseen bordes y si quisieramos visualizar las divisiones debemos habilitarlos

¿Estructura Semantica?

Con el surgimiento de HTML5 se redefinieron las etiquetas a utilizar dentro de una tabla, ya que la finalidad de las mismas, segun la w3c, es la presentacion de datos. Entre las etiquetas que se incorporaron estan <THEAD> para la cabecera, seccion donde se definen los titulos de la misma y en su interior en lugar de utililzar TD se implementan las etiquetas TH (Table Heading); <TBODY> para el contenido, en esta seccion se sigue utilizando la estructura clasica debido a que las etiquetas semanticas solamente generan una fila y en este caso de requiere de mucho mas contenido. Tambien se pueden utilzar celdas de titulo en caso de armar una tabla de referencias cruzadas. <TFOOT> funciona como complemento de los datos, generalmente para definir totales, detalles, promedios, etc. Tambien contener titulos, pero a diferencia de las dos anteriores, No es obligatoria.

Contenido Multimedia

Desde el surgimiento de HTML se pueden compartir todo tipo de archivos, especialmente contenido multimedia. Pero para la reproduccion del mismo, durante mucho tiempo fue necesario la implementacion de algun complemento o plugin de navegador, debido a que sin estos era practicamente imposible llevar a cabo dicha practica. Uno de los plugins mas conocidos fue el discontinuado Flash player.

¿Contenido Embebido?

Durante mucho tiempo la etiqueta <iframe> (Marco Flotante) se utilizo para la implementacion todo tipo de archivos, incluido el contenido multimedia, en un sitio web de forma asincronica . Basicamente es una ventana dentro de la pagina, que muestra un elemento externo, ya sea que corresponda a un archivo dentro del sitio o directamente otro sitio web dentro de nuestra pagina. Hoy en dia este tipo de contenido corresponde a una categoria de HTML5 conocida como Embebido (Embed), pero si necesitamos incorporar este tipo de elementos de manera local podemo utilizar los elementos que se mencionan mas adelante

¿Sonidos en la Web?

Como se menciono anteriormente la unica forma de insertar contenido multimedia en nuestra web, sin utilizar plugins, era mediante una etiqueta asicronica . Pero con la incorporacion del nuestro standard se agregaron nuevas etiquetas, entre ellas <AUDIO> que permite al navegador utilizar una API de reproduccion integrada que admite los formatos .mp3 - .wav - .ogg, pero tambien admite formatos de video como los que se mencionan mas adelante . Cabe destacar que para utilizar esta etiqueta sin JS, es necesario implementar el atributo controls ya que de lo contrario el reproductor no sera visible, al igual que la misma etiqueta.

¿Multimedia sin Plugins?

A diferencia de la etiqueta <iframe>, la etiqueta video carga su contenido conjuntamente con el resto de la pagina, pero a diferencia de metodos anteriores este no requiere de complementos que deban instalarse en el navegador. Entre los formatos admitidos se encuentran .mp4 - .webm - .ogv (en general formatos pensados para compartir contenido en la web) y a diferencia del elemento anterior no puede reproducir sonidos, sin cambiar el tipo de reproductor.
