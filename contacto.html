Formularios HTML

Son elementos que permiten una interaccion real (intercambio de datos) entre el cliente y el servidor, ya que a diferencia de otras etiquetas como <a>, donde el comportamiento esta predefinido, en el caso de los primeros el resultado dependera pura y exclusivamente de tipo y la manera en que se manipulado los datos. Para poder implementar dicho elemento, debemos utilizar la etiqueta <form> y solamente se tendra en cuenta al momento de enviar el formulario, los elementos que se encuentren dentro de este; pudiendo de esta manera manipular multiples cantidades de datos por separado.

¿Como funcionan?

A la hora de utilizar la etiqueta mencionada anteriormente, debemos indicar por lo menos 2 atributos, estos son action="..." & method="...". El primero indica que sucede al enviar el formulario y normalmente se coloca como valor un script en lenguaje de servidor como por ejemplo PHP, esto se debe a que es mucho mas seguro y confiable que el mismo servidor procese los datos y no el cliente. En cambio, el atributo method solamente recibe 2 valores, method="GET" que se utiliza cuando la prioridad la posee la respuesta del servidor y los datos del cliente, no son tan importantes, admite 8kb y la informacion es accesible desde la barra de direcciones, este metodo se considera poco seguro. El otro valor es method="POST" que se utiliza cuando la prioridad esta en los datos provistos por el usuario, se considera un metodo seguro debido a que encripta los datos antes de enviarlos, admite hasta 8mb y la informacion no es accesible desde la barra de direcciones.

¿Como estan compuestos?

Los formularios se componen de 2 tipos de elementos, por un lado estan las Etiquetas (Label) que permiten indicar con que datos se debe completar el mismo, facilita el trabajo de los screenreaders y permite a los motores de busquedas indexar el contenido del formulario segun la informacion que deba completarse. Otro de los elementos que encontramos son los Campos (Input, Textarea, Select) que permiten al usuario seleccionar o ingresar los datos que van a ser enviados a traves del mismo. En el caso de los campos es fundamental comprender que cuando se realiza el envio del formulario se envian los valores (value="..."), es decir que si el usuario escribe el valor esta definido por este, pero si no es asi, el valor debe estar previamente definido en la etiqueta.

Tablas HTML

Son elementos que permiten organizar la informacion de manera tabular y al igual que en cualquier otro ambito estan compuestas por filas y columnas, pero a diferencia de muchos programas, en HTML las mismas se crean a partir del modelo de cajas, es decir, utilizando etiquetas contenedoras y etiquetas de contenido. Podemos crear tablas siempre y cuando utilicemos la etiqueta <TABLE> para desglosar la estructura de la misma, debemos tener en cuenta que las mismas NO cuentan con borde de forma predeterminada y de no colocarlo, parecerian porciones de texto o elementos separados por espacios.

¿Estructura Clásica?

Desde sus comienzos, la estructura de las tablas carecian de contenido semantico, estando compuestas solamente por Filas (TR - Table Rows) y Celdas (TD - Table Division) reemplazando a la estructura convencional de filas y columnas. Es importante tener en cuenta que tanto las etiquetas <table> y <tr> cumplen la funcion de contenedores y la etiqueta <td> cumple la funcion de contenido. A diferencia del resto de las etiquetas de bloque, las tablas se adaptan a su contenido y no a su contenedor.

¿Estructura Semántica?

Con la llegada de HTML5 como nuevo estandar, se definieron nuevos elementos para complementar las etiquetas vistas anteriortemente, estos son THEAD (Cabecera de Tabla) cuya funcion en contener la fila de titulos y cada una de sus celdas es representada por un TH (Table Heading) en lugar de un TD; TBODY (Cuerpo de Tabla) el cual mantiene la estructura clasica y su funcion es presentar los datos, en caso de crear una tabla de referencias cruzadas tambien podemos implementar etiquetas TH en la columna correspondiente; y por ultimo TFOOT (Pie de tabla), siendo este ultimo opcional, contiene informacion complementaria a la tabla como por ejemplo totales, promedios, etc., tambien puede poseer titulos.

Contenido Multimedia

Desde que existe HTML, siempre se ha podido utilizar este tipo de elementos en las paginas. A lo largo de los el avance con el manejo de las etiquetas fue tal que ya no son necesarios complementos para poder reproducir este tipo de contenido, ya que los navegadores actuales poseen API's integradas que permiten realizar dichas acciones.

¿Contenido Embebido?

La etiqueta iframe originalmente se utilizo para insertar contenido de forma asincronica dentro de nuestra web, surge como una evolucion del frameset y hoy en dia se utiliza para embeber contenido de otras web, que puede no ser contenido html, dentro de nuestro sitio. Por ejemplo, mapas de Google, Videos de Youtube o Presentacion de SlideShare. Mas adelante veremos como implementar multimedia de forma nativa mediante los elementos audio y video

¿Sonidos en la Web?

Con la aparicion del nuevo estandar, se facilito la implementacion de archivos de audio, sin necesidad de utilizarlos de forma embebida, como se menciono anteriormente. Los formatos compatibles con dicho elemento son .mp3 - .wav - .ogg, pero como adicion tambien permite la reproduccion del formatos de video como los que se mencionan mas adelante. Para poder visualizar el reproductor de sonido, debemos especificar el atributo controls.

¿Multimedia sin Plugins?

Como se menciono anteriormente, en la actualidad ya no es necesario embeber contenido para poder reproducir elementos en nuestra web y mucho menos, la descarga de complementos de terceros. A diferencia del elemento audio que no cambia su apariencia cuando se reproduce otro tipo de formato que no sea de sonido, este elemento cambia de aspecto. Los formatos admitidos por la etiqueta <video> son .mp4 -.webm - .ogv
