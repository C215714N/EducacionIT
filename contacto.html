¿Que son los formularios?

Los formularios son una herramienta que permite el intercambio de datos entre el cliente y el servidor. Estos son la unica forma que tiene el usario de ingresar datos en un sitio web. Existen diferentes tipos de formularios segun la informacion que estos manipulen.

  • Busqueda: (Consulta)
  • Inscripcion: (Registro)
  • Acceso: (Inicio Sesion)
  • Consulta Web: (Contacto)

Los formularios se componen de 2 elementos principales, que son las etiquetas y los campos. El primero se utiliza para guiar al usuario, con respecto a la informacion que debe ingresar dentro del formulario y los ultimos, son los elementos donde el usuario va a ir ingresando o seleccionando los datos en cuestion.

Campos de Formulario

Existe diferentes etiquetas que podemos utilizar para la creacion de campos de formularios, segun lo que querramos implementar. Entre las mas utilizadas encontraremos <input/> , que corresponde a un campo de una sola linea; el cual posee diferente tipos, tambien contamos con los campos multilinea <textarea></textarea>; y por ultimo tenemos la etiqueta <select></select>, que lista las opciones que podemos elegir.

    Tipos de Datos (Input)
  • text: admite cualquier valor
  • number: admite solamente numeros
  • email: necesita el simbolo @ entre caracteres
  • date: permite seleccionar una fecha
  • password: oculta los caracteres escritos
  • radio: boton de opcion unica
  • checkbox: boton de opcion multiple
  • reset: reestablece los demas campos del formulario
  • submit: ejecuta la accion predeterminada

Acciones y Metodos

Cuando se valida un formulario, el proposito del mismo, es que la informacion sea enviada a un servidor y que este ultimo la procese para determinar que hacer con ella. El atributo method define como se va a trabajar con dicha informacion y admite 2 valores:

  • GET: se utiliza cuando necesitamos obtener informacion de un servidor, es un metodo poco seguro ya que no encripta los datos y admite hasta 8kb de informacion; esta ultima se puede visualizar en la barra de navegacion.
  • POST: se utiliza cuando los datos enviados al servidor son prioridad, por ende estos se encriptan y no se visualiza la informacion en otro lugar que no sean los respectivos campos; a diferencia del metodo anterior, este admite 8mb de informacion.
