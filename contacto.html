Formularios HTML

Son elementos que permiten una interaccion real (intercambio de datos) entre el cliente y el servidor, ya que la respuesta que brinde este ultimo dependera de la informacion completada en el mismo. Para poder utilizar este elemento debemos implementar la etiqueta < form> y solamente se enviaran los datos que se encuentren dentro de la misma.

¿Como funcionan?

Dentro de la etiqueta mencionada anteriormente debemos especificar 2 atributos, por un lado action="..." que permite indicar que pasara con los datos completados en el formulario al presionar el boton enviar y por el otro method="..." que indica como se procesaran los datos para su envio, es decir, el tipo de prioridad de los mismos.
En el caso de action="script.php" hoy en dia se suele implementar algun lenguaje de servidor, para que este ultimo trabaje con los datos enviados. En el cado de method="..." existen 2 valores posibles, estos son GET y POST.
GET se utiliza cuando la prioridad esta en la respuesta del servidor, admite hasta 8kb, es menos seguro y los datos son accesibles desde la barra de direcciones. Por otro lado POST se utiliza cuando la prioridad esta en los datos enviados por parte del cliente, admite 8mb, cifra los datos antes de enviarlos y los mismos no son accesibles desde la barra de direcciones.

¿Como estan compuestos?

Dentro de los formularios encontraremos 2 tipos de elementos, por un lado tenemos las Etiquetas (LABEL) que se utilizan para identificar que tipo de dato debemos ingresar o seleccionar en el formulario, y por el otro tenemos los Campos (INPUT, TEXTAREA, SELECT). En caso del primer elemento, este funciona para los screenreaders y para la busquedas mediante SEO y es obligatorio, el segundo por otro lado permite al usuario enviar datos al servidor y es necesario, existen diferentes tipos de campos segun el valor o tipo de dato a utilizar.

Tablas HTML

Son elementos que permiten organizar la informacion de manera tabular y a diferencia de otras etiquetas, el contenido siempre esta dispuesto en la posicion donde fue creada, es decir, que las mismas se adaptan al tamaño del contenido y no al tamaño del contenedor como el resto de las etiquetas de bloque. En comparacion con diferentes programas que usan tablas, HTML utiliza el modelo de cajas para la creacion de las mismas y como es comun con la mayoria de las etiquetas, de manera predeterminada estas no poseen borde.

¿Estructura Clásica?

Antiguamente era comun utilizar estos elementos para el diseño de las paginas web, pero con el surgmiento de CSS, fueron dejandose de lado para solamente organizar bloques de contenido. Para crearlas debemos utilizar la etiqueta <table> como contenedor principal y para crear el contenido debemos insertar primero las filas con la etiqueta <tr> (Table Row) y dentro de la misma la cantidad de celdas equivalente a la cantidad de columnas que necesitamos, utilizando la etiqueta <td>(Table Division)

Fila 1 Celda 1: 1 Fila 1 Celda 2: 2 Fila 1 Celda 3: 3
F2 C1: 4 F2 C2: 5 F2 C3: 6
F3C1: 7 F3C2: 8 F3C3: 9

¿Estructura Semantica?

Al igual que sucede con la estructura de una pagina, con la incorporacion del nuevo estandar, se agregaron nuevas etiquetas a las tablas para dotar de carga semantica a las mismas. Entre las etiquetas que se incorporaron estan <THEAD> que representa la cabecera de tabla, en su interior debe contener solamente titulos y estos se representan con la etiqueta <th> (Table Heading); <TBODY> que representa el cuerpo de la tabla y es la unica parte donde se sigue utilizando la estructura clasica, aunque si creamos una tabla de referencias cruzadas tambien puede poseer celdas de titulos. Por ultimo, <TFOOT> la cual es opcional, se utiliza como fila de detalles y al igual que el caso anterior, tambien puede poseer titulos.

Contenido Multimedia

¿Contenido Embebido?

Antiguamente la etiqueta <iframe> (Marco Flotante) se utilizaba para simplicar el diseño, ya que permite mostrar de manera asincronica otro tipo de contenido dentro de la misma, como pueden ser otras paginas web, documentos, contenido multimedia, etc. Hoy en dia se utiliza para embeber contenido (embed) dentro de nuestra pagina, generalmente compartido desde otra web como es el caso de los videos de Youtube, presentaciones de SlideShare o mapas de Google Maps. A diferencia de las etiquetas <audio> y <video> que se mencionan mas adelante, iframe se utilizo mucho durante la epoca donde no era comun reproducir de forma nativa contenido multimedia.

¿Sonidos en la Web?

A diferencia del elemento anterior, el elemento <audio> permite la incorporacion de un reproductor de sonidos integrado al navegador, los formatos compatibles con este elemento son .mp3 - .wav - .ogg, aunque tambien admite formatos de video como los que se mencionan mas adelante

¿Videos sin Plugins?

A diferencia del primer elemento, la etiqueta <video> permite integrar este tipo de contenido de forma nativa, sin necesidad de complementos de terceros como el clasico Flash Player que utilizaba YouTube en sus comienzos. Los formatos compatibles son .mp4 - .webm - .ogv pero a diferencia del elemento audio, este cambia a un reproductor diferente cuando se intenta reproducir otro formato que no sea video.
