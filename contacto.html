Formularios HTML

Son elementos que permiten una interaccion real (intercambio de datos) entre cliente y servidor, es decir que podemos seleccionar y/o completar con datos en la pagina y luego mediante una solicitud enviarlos al servidor para que los procese y resuelva que hacer con los mismos. Para poder implementar un formulario en necesario utilizar la etiqueta <form> que contendra los elementos del mismo y solamente se enviara la informacion colocada dentro de esta etiqueta, cabe destacar que una pagina puede poseer mas de un formulario y existen diferentes tipos.

¿Como funciona?

Los formularios requieren de 2 atributos para poder llevar a cabo el envio y procesamiento de la informacion. Por un lado tenemos el atributo action="..." que se utiliza para indicar que sucedera con la informacion una vez realizado el envio, antiguamente se utilizaba el valor mailto:usuario@dominio para abrir la aplicacion de correo predeterminada, pero hoy en dia se utiliza un script en lenguaje de servidor como por ejemplo PHP. Por otro lado, encontramos el atributo method="..." que indica como se realizara el envio del mismo, este atributo posee 2 valores; GET prioriza la respuesta del servidor para realizar peticiones, es menos seguro y admite hasta 8kb, y POST prioriza la seguiridad de los datos para enviarlos al servidor, ya que encripta los mismos y la informacion no es visible desde la barra de direcciones, ademas admite hasta 8mb.

¿Como esta compuesto?

Dentro de un formulario encontramos 2 tipos de elementos. Por un lado estan las Etiquetas (LABEL) que funcionan como rotulos, ya que indican al usuario que debe seleccionar o escribir en el formulario, permite a los motores de busqueda el indexado de los campos y a los lectores de pantalla leer dicha informacion. Por el otro estan los Campos (TEXTAREA / SELECT / INPUT) que representan las areas dentro del formulario donde podemos escribir o seleccionar opciones, los mismos poseen un atributo llamado value=".." el cual indica cuales son los datos que se estaran enviando (DINAMICO), asi como tambien un type="..." que indica cual es el tipo de dato permitido para el completado.

Tablas HTML

Las tablas son elementos que permiten organizar la informacion de manera tabular, al igual que cualquier en otro ambito poseen filas, columnas y celdas; pero a diferencia de la mayoria de los progamas que las utilizan, en HTML las mismas se crean a partir del modelo de cajas utilizando la etiqueta <TABLE> como contenedor principal de nuestra tabla. Las tablas de HTML a diferencia del resto de las etiquetas de bloque se adaptan al contenido en lugar de adaptarse a su contenedor, por lo tanto el tamaño de las mismas depende pura y exclusivamente de los datos; pero al igual que la mayoria de las etiquetas de bloque, tampoco poseen bordes.

Estructura Clasica

A la etiqueta mencionada anteriormente se le agregan dos elementos mas, uno de estos son las filas de la tabla que se representan con la etiqueta <TR> (TABLE ROW) que sirve como contenedor base para las celdas, y para poder crear estas ultimas debemos utilizar la etiqueta <TD> (TABLE DIVISION) que funciona como una etiqueta de contenido y a diferencia de la primera, se visualiza correctamente en la estructura. Antiguamente esta estructura se implementaba para el diseño de las paginas web, un ejemplo vigente de esto es la pagina de Space Jam que fue desarrollada en el mismo año que surge CSS.

Estructura Semantica

  • <THEAD> representa la cabecera de tabla, donde se colocan los titulos y por ende reemplaza a la primer fila, en su interior en lugar de colocar celdas, debemos colocar las etiqueta <TH> (Table Heading) para las celdas de encabezado.
  • <TBODY> corresponde al cuerpo de la tabla, la unica que mantiene la estructura clasica, pero tambien puede contener celdas de encabezado; especialmente si hablamos a una tabla de referencias cruzadas.
  • <TFOOT> hace referencia al pie de tabla o el contenido complementario de la misma, normalmente se lo utiliza para representar valores que esten relacionados con el contenido del TBODY como totales, promedios, etc.

Contenido Multimedia

A lo largo de los años existieron multiples formas de implementar contenido multimedia en nuestros sitios web, ya sea utilizando aplicaciones de terceros como complementos o plugins para los navegadores web, hasta API de reproduccion integradas para maximizar la compatibilidad con estos ultimos.

¿Contenido Embebido?

Una de las primeras etiquetas que permitio la integracion de contenido externo en nuestra web fue <iframe> (Marco Flotante) que mostraba de manera asincronica, en una ventana flotante, el contenido de la direccion especificada, la misma podia ser otra pagina dentro del mismo sitio, asi como tambien una pagina o un archivo externo. Hoy en dia esta etiqueta pertenece a una categoria web conocia como EMBED (embebido) que se utiliza para definir a todos los elementos que pertenecen a otro ambito pero pueden ser utilizados dentro de HTML, por lo que la reproduccion de contenido multimedia queda relegada exclusivamente para las etiquetas <audio> y <video> que se veran mas adelante.

¿Sonidos en la Web?

Una forma de utilizar sonidos en una pagina web es mediante la etiqueta <audio>, que a diferencia del elemento anterior necesita que el contenido se encuentre alojando en nuestro hosting. Dentro de los formatos compatibles con esta etiqueta se encuentran .mp3 - .wav - .ogg, aunque tambien admite los mismos formatos que la etiqueta <video>, que se detallan mas adelante, en este caso solamente toma la pista de audio de estos ultimos. Cabe destacar que para que se visualice un reproductor es necesitario utilizar el atributo controls

¿Videos sin plugins?

La etiqueta <video> es la solucion a las limitaciones que trae el uso del elemento iframe mencionado anteriormente, permite la reproduccion nativa de los formatos de video .mp4 - .webm - .ogv los cuales estan optimizamos para web debido a que son ligeros, por lo tanto No es necesario descargar ningun complemento para su reproduccion. A diferencia del elemento anterior, si bien se pueden reproducir formatos de audio, el aspecto del reproductor cambia radicalmente, pero la etiqueta sigue siendo la misma. Otro detalle a destacar es que no se precisa utilizar el atributo controls para visualizar la etiqueta, al igual que se admite el atributo autoplay siempre y cuando este acompañado del atributo muted.
