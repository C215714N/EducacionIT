Formularios HTML

¿Para que sirven?

Son elementos que permiten una interaccion real (intercambio de datos) entre cliente/servidor debido a que permiten realizar peticiones al servidor y este ultimo envia una respuesta que varia dependiento del tipo de formulario completado. Para poder implementarlos es necesario utilizar la etiqueta <FORM>, que debe contener todos los elementos se veran a continuacion.

  • Busqueda - Consulta Query
  • Inscripcion - Registro
  • Acceso - Inicio Sesion
  • Contacto - Consulta Web

¿Como funcionan?

A la etiqueta mencionada anteriormente debemos colocarle por lo menos 2 atributos necesarios para su funcionamiento. Por un lado se encuentra action="..." que permite definir que sucedera con los datos al realizar el envio del formulario, antiguamente se utilizaba el valor mailto:correo pero en la actualidad se implementa un script en lenguaje de servidor (ej: PHP) para garantizar la recepcion del mensaje. Por otro lado esta el atributo method="..." que indica como se realizara el envio y requiere de un valor, que puede ser GET en caso que la respuesta del servidor sea prioritaria o POST en caso que los datos del usuario sean lo mas relevante.

  • GET: Metodo menos seguro ya que los datos son accesibles desde la barra de navegacion, pero es mas rapido debido a que realiza el envio en texto plano.
  • POST: Metodo mas seguro ya que encripta los datos que se envian y los mismos no estan visibles, pero es mas lento debido a que debe encriptar y desencriptar los datos.

¿Que elementos los componen?

Dentro de los mismos utilizaremos 2 tipos de elementos , por un lado estan las etiquetas (LABEL) que funcionan como rotulos que indican al usuario que tipo de datos debe ingresar/seleccionar y ademas facilitan la accesibilidad. Por el otro lado estan los campos (INPUT / TEXTAREA / SELECT) que permiten al usuario seleccionar/completar con datos el formulario y los cuales contendran dicha informacion, estos elementos cuentan con 2 atributos type="..." que indica el tipo de dato utilizado y value="..." que es el dato en si mismo.

Tablas HTML

¿Cuando utilizarlas?

Son elementos que permiten organizar la informacion de manera tabular, deben utilizarse cuando sea necesario respetar una estructura de elementos (datos) que debe mantener su organizacion. A diferencia de las demas etiquetas de bloque, las tablas se adaptan al tamaño de su contenido y se crean respetando el modelo de cajas (contenedor y contenido), si bien posee columnas, filas y celdas, debemos crear la tabla <TABLE>, luego las filas <TR> y por ultimo las celdas <TD>, aunque esto aplica solamente para la estructura clasica y no la estructura semantica detallada mas adelante.

¿Elementos semanticos?

Como se menciono anteriormente, las tablas se crean respetando el modelo de cajas, pero no por esto quiere decir que no sean semanticas. Para poder definir la funcion que cumple cada parte de la misma, la w3c elaboro 3 elementos estructurales para facilitar el analisis de las tablas por parte de los motores de busqueda, los mismos en la mayoria de los casos reemplazan a las filas <tr> a excepcion del cuerpo que mantiene dicha estrucuta.

  • THEAD: Cabecera de tabla que contiene los encabezados y las celdas en su interior deben ser de titulo <th> (table heading) Debido a que indica los datos que van a ser representados
  • TBODY: Cuerpo de tabla que contiene los datos y unica parte que mantiene la estructura clasica de tabla, aunque segun el tipo de tabla, por ejemplo referencias cruzadas, tambien puede poseer encabezados <th>
  • TFOOT: Pie de tabla que contiene el detalle o datos complementarios al cuerpo como totales, promedios, etc.. Es opcional y puede poseer tanto celdas de contenido <td> como encabezados <th>
