Formularios HTML

¿Para que sirven?

Son herramientas que sirven para intercambiar datos entre cliente / servidor, permiten una interaccion real entre el usuario y el sitio. Segun el tipo de formulario, lo que suceda con los datos dependera del servidor al momento de recibirlos y por ende la respuesta al envio no siempre sera la misma. Para implementarlos debemos colocar la etiqueta <form>con determinados atributos, y en su interior, etiquetas que se mencionan mas adelante, segun lo que deba colocar el usuario.

  • Inscripcion - Registro
  • Acceso - Inicio de Sesion
  • Contacto - Consulta Web

¿Como funcionan?

La etiqueta mencionada anteriormente posee al menos 2 atributos que debemos especificar. Por un lado, tenemos el action="..." que determina que sucedera con los datos una vez que se haya enviado el formulario, normalmente se coloca en su interior un script en lenguaje de servidor(ej: php) para que este resuelva que hacer con los datos. Por otro lado encontramos el atributo method="..." que indica cual es el tipo de prioridad y como se debe realizar el envio de los datos.

  • GET: Se utiliza para obtener datos del servidor y por lo tanto la prioridad radica en la respuesta del mismo. La comunicacion se realiza en texto plano y por ende es mucho mas rapida, pero menos segura.
  • POST: Se utiliza para el envio de datos desde el cliente y por lo tanto la prioridad radica en el envio. La comunicacion se encripta y por lo tanto se lo considera un metodo mas seguro, pero mucho mas lento.

¿Que elementos los componen?

Dentro de un formulario encontraremos 2 tipos de elementos, las etiquetas(LABEL) que funcionan a modo de rotulos, identificando que datos debe ingresar / seleccionar el usuario y facilitan la accesibilidad; por otro lado los Campos(INPUT / TEXTAREA / SELECT) permiten al usuario en ingreso / seleccion de los datos, teniendo en cuenta que poseen 2 atributos necesarios que son type="..." para indicar el tipo de dato a completar y value="..." que representa el valor ingresado/seleccionado que va a ser enviado.

Tablas HTML

¿Cuando utilizarlas?

Son elementos que permite organizar la informacion de manera tabular, y se deben utilizar cuando necesitamos mostrar datos en pantalla de manera correlativa y respetando cierta estructura. Las mismas se crean respetando el modelo de cajas, donde se utiliza etiquetas como contenedores, por ejemplo: <table> y <tr> y etiquetas como contenido, por ejemplo: <td>.

¿Poseen carga semantica?

Con la aparicion de HTML5, las tablas tuvieron un lavado de cara, debido a que se les agregaron nuevos elementos que permiten definir la funcion que cumple cada parte dentro de la misma. Estos complementan a los mencionados anteriormente.

  • THEAD: Cabecera de tabla, encargada de contener los titulos de la misma, que se identifican con la etiqueta <th>
  • TBODY: Cuerpo de tabla, contiene la informacion de la misma, aqui se mantiene la estructura clasica, aunque tambien puede poseer titulos, por ejemplo: tabla de referencias cruzadas
  • TFOOT: Pie de tabla, parte opcional que muestra informacion complemetaria a los datos representados en la tabla, por ejemplo: totales, promedios, max, min, etc.
