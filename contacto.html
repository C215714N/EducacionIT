Formularios HTML

¿Para que sirven?

Son una herramienta que permite la interaccion real, entre el cliente y el servidor, ya que a diferencia de otros elementos que tienen un comportamiento predeterminado, la informacion almacenada en estos dependera pura y exclusivamente del usuario. En la mayoria de los casos dependeremos de un lenguaje de backend para poder validar la informacion completada en los mismos, ya que los datos seran enviados a un servidor para que este determine que hacer con dicha informacion.

Dentro de un formulario vamos a encontrar diferentes elementos, que los podemos clasificar en Etiquetas (label) y Campos (select - textarea - input). El primero se utiliza para identificar que tipo de informacion debemos escribir o seleccionar del formulario y, por otro lado, el segundo es un conjunto de elementos de diferente tipo que nos permite completar con datos o seleccionar informacion previamente establecida.

  • Input TEXTO
  • Input NUMERO
  • input FECHA
  • input CORREO
  • input CONTRASEÑA
  • opcion UNICA
  • opcion MULTIPLE

¿Como funcionan?

Para poder enviar un formulario existen dos atributos fundamentales, que le indican al navegador que se debe hacer y de que manera, estos son action y method. Para saber que valor debemos utilizar, necesitamos tener en cuenta el tipo de formulario que vamos a implementar en nuestra pagina.

  • Buscador (Consulta Query)
  • Inscripcion (Registro)
  • Acceso (Inicio Sesion)
  • Consulta Web (Contacto)

Antiguamente, antes de la llegada de la banda ancha, era comun que los usuarios utilizaran aplicaciones de correo para descargar el mismo en las computadoras y por lo tanto los formularios antiguos utilizaban action="mailto:cristiandracedo@hotmail.com", pero debido a que hoy en dia esto es poco impractico, se utiliza un script en lenguaje de servidor. Por otro lado, exiten dos valores posibles para el method que se caracterizan por lo siguiente:

  • GET
    • Metodo menos seguro (No encripta los datos enviados)
    • Se implementa cuando la respuesta es lo mas importante y no tanto los datos
    • La informacion es accesible en todo momento
    • Admite hasta 8kb
  • POST
    • Metodo mas seguro (Cifra los datos al enviarse)
    • La informacion no es accesible directamente desde el navegador
    • Los datos enviados son la prioridad y no tanto la respuesta (que puede o no existir)
    • Admite hasta 8Mb

Tablas HTML

¿Que son?

Son elementos que permiten organizar la informacion de manera tabular, y al igual que en cualquier otro ambito, estas se componen de filas y columnas, pero no funcionan de la misma que en el resto de los programas que las utilizan. Las tablas en HTML respetan la estructura de cajas, utilizando como base las siguientes etiquetas: <table> - <tr> - <td> de forma tal que debemos utilizar un contenedor TABLE donde insertaremos el contenido, por cada fila debemos utilizar un contenedor TR (Table Row) para indicar donde deben alojarse las celdas, las cuales estaran marcadas con la etiqueta de contenido TD (Table Division).

A1 A2 A3
B1 B2 B3

¿Nuevo paradigma?

A partir de la standarizacion de HTML5, se incorporaron a estructura clasica de las tablas etiquetas con carga semantica. La razon por la que se implemento esto, fue para redefinir y reforzar la estructura y el contenido de las mismas, por un lado para mejorar la busqueda de contenidos por parte de los motores de busqueda y para facilitar el diseño de las mismas a los desarrolladores.
<thead> Cabecera de tabla que reemplaza a la primer fila y contiene los titulos de las columnas, en su interior reemplazamos las etiquetas <td> por etiquetas de encabezado <th>
<tbody> Cuerpo de Tabla, es la unica instancia donde se sigue utilizando la estructura clasica para los datos, pero contenida.
<tfoot> Pie de Tabla, etiqueta opcional que complementa la informacion desplegada en el cuerpo de la tabla, generalmente se suelen colocar detalles o totales.

Contenido Multimedia

¿Antes de HTML5?

Antiguamente si necesitabamos incorporar un contenido de manera asicronica en nuestra pagina web, contabamos con una etiqueta llamada <iframe> que nos permitia mostrar dentro de un marco flotante el contenido de otra pagina o sitio. Hoy en dia se sigue utilizando esta etiqueta, pero entra en la categoria embed que define a todos los elementos que no pertecen al universo HTML, pero se pueden utilizan en este.

En serio ¿Sin Plugins?

Con el standard HTML5 los navegadores utilizan una API que permite la reproduccion nativa de elementos multimedia, incorporando dos nuevas etiquetas <audio&> y <video>, sin necesidad de instalar complementos externos a este. Si bien, la mayoria de los navegadores es compatible con estos elementos, versiones antiguas de Internet Explorer no lo soportan; pero aquellos navegadores que si lo hacen admiten los siguientes formatos de Audio .mp3 - .wav - ogg y tambien los formatos de Video .mp4 - .avi - .ogv.
