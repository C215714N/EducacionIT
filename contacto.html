Formularios HTML

Son elementos que permiten una interaccion real (intercambio de datos) entre cliente y servidor, es decir que el navegador envia una solicitud/peticion y el servidor donde se aloja el sitio determina que debe hacer con la misma. Una pagina web puede poseer mas de un formulario, pero para poder distinguir que datos se deben enviar, el contenido a procesarse debe encontrarse dentro de una etiqueta conocida como <form>

多Como funcionan?

Dentro de la etiqueta mencionada anteriormente debemos colocar por lo menos 2 atributos. El primero es action="..." que determina que se debe hacer con los datos una vez enviado el formulario, antiguamente se utilizaba el valor mailto:usuario@servidor pero hoy en dia se utiliza un script en lenguaje de servidor como PHP, para asegurarse que los datos se trabajen correctamente. El otro atributo es method="..." que indica como se procesa la informacion del formulario, el mismo recibe 2 valores y estos son GET para realizar peticiones al servidor, priorizando la respuesta y por otro lado POST que realiza envios de datos para que servidor defina que hacer con ellos, la prioridad se encuentra en la informacion provista por el usuario.

  • GET
    • menos seguro (Prioridad Servidor)
    • la informacion es accesible desde la barra de direcciones
    • admite hasta 8kb
    • Buscadores - Consulta Query
  • POST
    • cifra los datos (Prioridad Cliente)
    • los datos no son accesibles fuera del formulario
    • admite hasta 8mb
    • Registro - Inscripcion
    • Acceso - Inicio sesion
    • Consulta web - contacto

多Como estan compuestos?

Los formularios poseen 2 tipos de elementos, estos son las etiquetas (LABEL) que funcionan como rotulos que indican al usuario cuales son los datos que debe ingresar o seleccionar, tambien permiten el indexado del contenido y ser leidos por parte de los screenreaders. Por otro lado encontramos los campos de formulario (INPUT / TEXTAREA / SELECT ) en los cuales el usuario va a escribir o seleccionar datos, estos poseen 2 atributos escenciales; por un lado esta el value="..." que representa la informacion escrita o seleccionada y luego esta el type="..." que indica el valor admitido por el campo de formulario.

Tablas HTML

Las tablas se componen de filas y columnas, pero a diferencia de cualquier otro programa html pemite su creacion basandose en el modelo de cajas, es decir utilizando la etiqueta contenedora <TABLE> y en su interior colocando el contenido distribuido en filas. A diferencia de la mayoria de las etiquetas de bloque, las tablas se adaptan a su contenidoen lugar de adaptarse a su contenedor. Si bien hoy en dia se utilizar para presentar informacion de manear tabular, antiguamente se utilizaban para el dise単o de las paginas hasta que empezo a implementar CSS como estandar.

Estructura Clasica

Para colocar la informacion dentro de las tablas, debemos insertar las etiquetas de fila <TR> (table row) para organizar donde se colocaran las celdas mediante la etiqueta <td> (table division) que funciona como etiqueta de contenido. Por defecto las tablas no poseen bordes y las celdas presentan lo que se conoce como cellspacing

Estructura Semantica

  • THEAD: fila de titulos, se utiliza para definir los encabezados de la tabla y en su interior debe poseer celdas de titulo utilizando la etiqueta th
  • TBODY: seccion de contenido, es donde se utiliza la estructura clasica ya que el mismo estara dividido en varias filas, aunque tambien se pueden utilizar titulos como en el caso de una tabla de refencias cruzadas
  • TFOOT: pie de tabla, o seccion de detalle se utiliza para ubicar la informacion complementaria a la tabla, como por ejemplo totales, promedios, etc.. Tambien puede presentar titulos aunque a diferencia de las anteriores es opcional.
