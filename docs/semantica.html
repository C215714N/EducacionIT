Estructura semantica del BODY en HTML

HEADER: Cabecera de Pagina.
Utilizada para etiquetas de titulo como <H1></H1>, imagenes a modo de banner o logo de la empresa en cuestion e incluso algun que otro slider.

NAV: Barra de Navegacion.
Aqui colocaremos los enlaces principales para el acceso a paginas en nuestro sitio, utilizando la etiqueta <a></a>, generalmente dentro de un <li></li> de una lista.

MAIN: Elemento Principal
Esta seccion esta destinada para el contenido principal de nuestra pagina, puede servir tanto como contenedor, asi como tambien como un elemento unico.

ARTICLE: Contenido Independiente.
Si el contenido de nuestra pagina web puede funcionar tanto dentro como fuera de la misma, debe colocarse en dicha etiqueta.

SECTION: Contenido Contextualizada.
Aqui colocaremos varios elementos que guarden una relacion directa con las demas secciones de nuestra pagina web, sin los cuales nuestro contenido no tendria fundamentacion alguna.

ASIDE: Contenido Tangencial.
Generalmente utilizada a modo de barra de contenido lateral, esta etiqueta es utilizara para aquel contenido que no se relaciona con el contenido principal, pero guarda una relacion tematica con el mismo.

FOOTER: Pie de Pagina.
Al final de nuestro documento, se suele colocar informacion sobre el desarrollo de la misma, como datos de © Derechos de Autor, Propiedad intelectual, datos de contacto utilizando etiquetas como <name></name>,<tel></tel>,<address></address>, o un formulario de contacto con la etiqueta <form></form>.

A diferencia del modelo de estructura en versiones anteriores de HTML, donde se realizaba un uso abusivo del <DIV></DIV>, las etiquetas Semanticas fueron pensadas para optimizar tanto del dise침o como para reforzar la estructura de nuestra pagina web.
Como su nombre lo indica, estas etiquetas alteran el significado de las expresiones segun como las utilicemos y en base a esto los Bots de los motores de busquedas, mediante algoritmos categorizaran e indexaran el contenido de nuestro sitio.
