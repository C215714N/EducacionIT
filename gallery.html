Reglas de Nomenclatura

  • Evitar el uso de espacios o caracteres especiales ya que el navegador reemplazara los mismos y no podra encontrar los archivos. por ejemplo, si nuestra imagen se llama "foto 01.jpg" el navegador interpretara "foto%2001.jpg" no pudiendo encontrar la misma, por eso es aconsejable utilizar guiones (-_) dando como resultado un nombre como "foto_01.jpg".
  • Utilizar nombres descriptivos como por ejemplo "background.jpg" o "perritos_jugando.jpg" para facilitar la implementacion de los archivos. Evitar nombres genericos como "foto_01.jpg" o automatizados como "2022-11-01-1055100.jpg" ya que demasia se dificulta el reconocimiento de los mismos.
  • Utilizar solamente minusculas para reducir el margen de error debido a que los servidores son caseSensitive y por lo tanto "background.jpg", "Background.jpg" y "BACKGROUND.jpg" se consideran archivos diferentes.

Estructura Semantica

Son equitetas que se utilizan para definir cuales son las partes que componen la pagina, si bien algunas son obligatorias, la implementacion de las mismas dependera del tipo de sitio que estemos desarrollando y su contenido.

  • header cabecera de pagina, se utiliza para identificar el contenido introductorio, por ejemplo: titulos
  • nav navegacion principal, se utiliza para guiar al usuario por las paginas principales del sitio (links mas importantes)
  • main contenido principal, se utiliza para indicar donde se encuentra el contenido mas importante o relevante para el usuario
  • section contenido contextualizado, se utiliza para indicar que dicha seccion se corresponde con otros elementos de la pagina.
  • article contenido independiente, se utiliza para indicar que dicho contenido funciona tanto dentro como fuera de la pagina.
  • aside contenido tangencial, se utiliza para indicar que dicho contenido se relaciona indirectamente con sus hermanos.
  • footercontenido complementario, se utiliza para indicar al usuario que otros sitios puede visitar, dar informacion de contacto o un mapa del sitio.
