Nuevas Caracteristicas

Incorporaciones al estandar

Con la aparicion de html5 se agregaron nuevas etiquetas al lenguaje para facilitar a los motores de busqueda la identificacion de cierto tipo de contenido, que anteriormente se implementaban de forma visual. Entre estos se encuentran API's de reproduccion de contenido multimedia, API's que permiten dibujar sobre el navegador, e incluso elementos que facilitan el analisis estadistico.

Elementos estadisticos

Para la representacion de valores numericos en diferentes circunstancias existen las etiquetas <progress> que se utiliza para representar el avance en una tarea cuando esta aun no fue finalizada, por ejemplo: la realizacion de una capacitacion; y <meter> que se utiliza para la comparacion de valores cuando una actividad fue finalizada, por ejemplo: el resultado de un examen.

  • Progress
    • value = undefined
    • value = 12.5; max = 100
    • value = javascript
  • Meter
    • value < low < high < optimum
    • low < value < high < optimum
    • low < high < value < optimum

Contenido Multimedia

Uno de los elementos incorporados con la ultima version de HTML5 fueron las etiquetas <audio> y <video> que permiten la reproduccion nativa de los formatos .mp3 - .wav - .ogg y .mp4 - .webm - .ogv respectivamente. Estas permiten acceder a la API de reproduccion, la cual podemos visualizar utilizando el atributo controls, incluso podemos hacer que estos elementos se reproduzcan automaticamente en bucle si utilizamos los atributos autoplay muted loop.

Herramientas de Dibujo

Los navegadores actuales incluyen herramientas que nos permiten realizar dibujos dentro de la pagina, estas son los elementos <SVG> (scalable vector graphics) para la realizacion de imagenes creadas matematicamente a partir de lineas y curvas utilizando el lenguaje XML; y <canvas> (Mapa de Bits) para la creacion de imagenes rasters, utilizando el lenguaje JavaScript para la creacion de los mismos.

Preprocesadores

Pseudolenguajes de CSS

Es un programa que permite generar CSS a partir de una sintaxis única, estas herramientas disponen de utilidades que no están en el lenguaje CSS y, por tanto, no las interpretan los navegadores directamente. Utilizando un preprocesador conseguimos economizar tiempo, ya que escribimos menos código. Además, su mantenimiento es fácil y resulta más comprensible, debido básicamente a su optimizada organización. Por el contrario, utilizar un preprocesador implica aprender un nuevo lenguaje y escribir código poco común, que debe ser compilado en CSS y que afecta al flujo de desarrollo. Este código, que no entiende el navegador, deberemos traducirlo antes de llevar un sitio web a producción.

LESS (Leaner Styles Sheets)

Preprocesador que utiliza Javascript como lenguaje principal para la compilacion de la hoja de estilos, podemos instalarlo mediante NodeJS utilizando el comando npm i -g less. Cabe destacar que dicha herramienta posee algunas ventajas del lenguaje como la asincronicidad y la capacidad de invocar funciones o metodos; y para compilar nuestros archivos debemos utilizar el comando lessc "input.less" "output.css"

Uno de los inconvenientes que presenta el paquete original de LESS radica en la falta de un metodo de observacion, es decir, que debemos compilar cada vez que realizamos cambios. Una solucion a esto es el paquete less-watch-compiler que podemos instalar con NodeJS y para compilar debemos utilizar el comando less-watch-compiler "input/" "output/" "file.less"

SASS (Syntactically Awesome Styles Sheets)

Preprocesador que utiliza Ruby como lenguaje principal para realizar la compilacion, una de las principales diferencias con respecto a LESS es que no permite el llamado de metodos si estos no fueron declarados previamente, es decir, que el orden de los elementos es fundamental. Tambien se caracteriza por diferencia cada tipo de elemento mediante su sintaxis. SASS posee dos tipos de sintaxis, sass que se maneja mediante tabulaciones y scss que es mas similar a css.

Para instalarlo debemos ejecutar desde la consola el comando npm i -g sass y para realizar la compilacion de los archivos utilizamos el comando sass --watch "input.scss" "output.css" permitiendonos realizar un seguimiento al mismo. En caso de presentarse algun problema en la compilacion, este permanecera visible dentro del sitio que utilice la hoja de estilos compilada con errores.

Transiciones y Transformaciones

La ultima version de CSS incorpora caracteristicas que nos permiten realizar cambios en la apariencia de las etiquetas de igual manera que algunos programas de diseño como Adobe illustrator, Inkscape o CorelDRAW, y a diferencia de otros atributos, estos cambios afectan a los elementos contenidos por dichas etiquetas. Ademas ofrece la posibilidad suavizar los efectos mediante transiciones que funcionan cuando realizamos cambios utilizando :pseudoclases.

translate(x,y)
scale(units)
rotate(deg)
skewX(deg)
skewY(deg)

Si bien podemos definir transformaciones en 2 dimensiones sin necesidad de ningun referente, para las transformaciones en 3 dimensiones la situacion es diferente, ya que si no definimos los atributos perspective y perspective-origin, que se utilizan para indicar la distancia y posicion con respecto al punto de fuga, seguiran siendo bidimensionales.

rotateX(deg)
rotateY(deg)
translateZ(d)
perspective

Animaciones CSS

Fotogramas clave

Al igual que algunos programas de diseño como Adobe Animate (antiguamente conocido como FLASH) podemos realizar animaciones en nuestra pagina web mediante la utilizacion de fotogramas claves donde indicamos cual debe ser la apariencia o cambios de nuestros elementos durante la misma, siendo el navegador el responsable de llenar los espacios intermedios.

Timing-Function

Esta caracteristica controla el modo en el que sucede la animacion y su valor predeterminado es ease. En caso de necesitar algun valor diferente a los que nos ofrece CSS, podemos personalizar la transicion utilizando el valor cubic-bezier(x1,y1,x2,y2) donde la "x" representa el tiempo e "y" la progresion, pudiendo realizar diferentes combinaciones con una sola animacion.

ease
ease-in
ease-out
ease-in-out
linear
cubic-bezier

Direction

Como si se tratara de una pelicula, tambien podemos controlar la direccion de la misma, a diferencia del atributo transition que rellena los espacios. El valor predeterminado es normal que reproduce la animacion del from{...} al to{...} y en caso de necesitar que sea a la inversa podemos utilizar reverse.

Podemos hacer que nuestra animacion alterne entre los diferentes valores, pero para ello es necesario que definamos el atributo animation-iteration-count con un valor mayor a 1 para que esto pueda ser aplicado sin problemas, ya que de lo contrario, se reproducira en el primer sentido y luego la misma finalizara.

normal
reverse
alternate
alternate-reverse

Percentages

En caso de necesitar una mayor cantidad de cambios durante la animacion, podemos indicar cuales son los fotogramas clave utilizando porcentajes, pero la misma siempre debe iniciar con 0%{...} y finalizar con 100%{...} ya que de lo contrario se interrumpira abruptamente.

Ademas podemos agregar alguna funcionalidad adicional con el atributo animation-play-state el cual define es estado de la animacion, los cuales son running y paused para que el usuario presione algun boton que cambie dicho valor al elemento seleccionado.
