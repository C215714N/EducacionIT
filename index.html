Introduccion a la Maquetacion

Elementos Semanticos

  • header representa la cabecera de pagina, donde colocamos el contenido introductorio. Entre las etiquetas que podemos encontrar en su interior estan los H1, algunos links e incluso la navegacion; aunque ocasionalmente tambien podemos apreciar alguna galeria. Esta etiqueta responde a la pregunta ¿quienes somos?
    • nav representa la navegacion principal, es donde debemos colocar los links mas importantes y por lo tanto, tenemos de definir cuales son. Existe una regla que determina "si para acceder a una pagina necesito hacer mas de 3 clicks, no merece la pena colocarla como parte de la navegacion principal" asi como tambien otra que dice "si todo es importante, nada es importante"
  • main representa al contenido principal de la pagina, toda informacion relevante se va a encontrar aqui segmentada en grupos segun el tipo de contenido, en caso de ser necesario. Aunque solamente con esta etiqueta es mas que suficiente para el indexado del contenido, dependiendo de la complejidad del mismo deberemos utilzar otras. Esta etiqueta responde a la pregunta ¿que le interesa al usuario?
    • article representa un articulo o contenido independiente dentro del sitio, es decir que funciona tanto dentro como fuera de la pagina, un ejemplo claro de este tipo de etiquetas son las noticias que poseen una estructura definida y se comprenden incluso fuera de su contexto original.
    • section representa una seccion o parte de un contenido dentro del contexto, el necesario que este dentro del sitio con los elementos con los que se relaciona porque al utilizarlo fuera de la pagina se pierde nocion del contenido, comunmente identificamos estas etiquetas por textos como "al igual que se menciona en el apartado anterior","se mencionara mas adelante" o dando por sentado ciertos temas.
    • aside representa el contenido lateral o tangencial, es decir, aquel que guarda una relacion indirecta con sus hermanos, normalmente se lo utiliza para publicidad o sugerencias dentro del mismo sitio y se le suele dar estilo de barra lateral, aunque al igual que el resto de las etiquetas semanticas, es de bloque.
  • footer representa el pie de pagina, aqui colocamos el contenido de despedida, como por ejemplo informacion del autor, copyright, redes sociales, mapa del sitio y datos o un formulario de contacto . El contenido de esta etiqueta es relevante para el usuario despues del contenido principal, engloba cualquier dato de interes y responde a la pregunta ¿que mas puede visitar?

Jerarquias de Selectores

  • tag selector selector menos especifico, por lo tanto con menor prioridad a la hora de aplicar las reglas, ya que apunta a todas las etiquetas (general). En el ejemplo se puede observar que se apunta a todas las etiquetas header y a todos los h1 que se encuentren dentro de header.
    • header{...}
    • header h1{..}
  • class selector selector con mayor especificidad, ya que apunta a todas las etiquetas que posean la clase seleccionada, indistintamente si estas pertenecen, o no, al mismo tipo (grupal). En este caso se apunta a todas las etiquetas con Clase header y a todos los h1 dentro de un elemento con clase header.
    • .header{...}
    • .header h1{...}
  • id selector selector mas especifico y por lo tanto con mayor prioridad, debido a que apunta a la etiqueta con el Id espeficado y como no debe existir mas de una con el mismo id es lo considera unico (individual). En esta situacion se apunta al elemento con id header y a todos los h1 dentro de dicha etiqueta.
    • #header{...}
    • #header h1{...}

Introduccion Atributos CSS

  • box-sizing: Indica al navegador que partes de las etiquetas se deben tomar en cuenta para calcular el tamaño final de las mismas, el valor predeterminado es content-box que contempla solamente el tamaño del contenido y luego debemos sumar el tamaño de padding y border para saber cual es el tamaño final de nuestro elemento. Por el contrario border-box contempla todo esto al definir el width.
    • Content-Box: content-width = width || final-width = width + padding + border
    • Border-Box: content-width = width - padding - border || final-width = width

Planificacion del Diseño

Unidades de Medida

  • Absolutas: son aquellas que, independientemente del tamaño de la pantalla donde se visualice el contenido, siempre representan lo mismo.
    • tamaños: cm - mm(0.1cm) - in(2.54cm) - 1pc (0.6in)
    • tipográficas: pt(1/72in - 0.35mm)
    • híbrida: px(depende de la resolucion de pantalla) - desktop 1px = 1px || phones 1px = 3px
  • Relativas: son aquellas que dependen exclusivamente de los valores que posea el elemento padre para determinar el aspecto de las etiquetas.
    • porcentaje: % de tamaño del elemento padre (contenedor)
    • tipográficas: em(ancho "M") - ex(alto "x") - ch(alto "0") - relativas al elemento padre
    • híbrida: rem(root em - depende del valor tipografico de HTML) - predeterminado = 16px

Propiedades de fondo

  • Background-image: permite definir una imagen de fondo utilizando el protocolo url(), la direccion debe estar escrita solamente con barras diagonales "/" y se posiciona por encima del color de fondo
  • Background-size: define el tamaño de la imagen de fondo utilizada, podemos utilizar valores en px, %, rem e incluso valores especiales hechos para las imagenes svg como cover y contain.
  • Background-position define la posicion inicial de la imagen de fondo, y es a partir de donde se van a crear las diferentes copias en caso de ser necesario, la posicion predeterminada es la esquina superior izquierda.Si bien se pueden utilizar valores en px, %, rem, tambien podemos utilizar palabras reservadas como top , bottom para el eje vertical y left, right para el eje horizontal; por ultimo center que funciona para ambos
  • Background-repeat: define como se va repetir la imagen de fondo, el valor predeterminado es repeat, pero podemos evitarlo o especificar sobre que eje se van a repetir las imagenes.
  • Background-attachment: permite definir la imagen de fondo como fija, para generar un efecto de movimiento, tambien conocido como efector parallax

Introduccion a Flexbox

Distribucion de elementos (justify-content)

  • flex-start: Define que todos los elementos se distribuyan desde el comienzo del flex (izquierda) hasta el final, es el valor predeterminado
  • flex-end: Define que todos los elementos se distribuyan desde final del flex (derecha), hasta el comienzo del flex, respetando el orden de aparicion
  • center: Distribuye los elementos de manera centrada en la columna (horizontal)
  • space-around: Distribuye las etiquetas dentro del contenedor con la misma distancia de margen para cada una, pero las distancia entre etiquetas se suman. El espacio entre etiquetas es mayor al espacio en los extremos
  • space-between: Distribuye de extremo a extremo las etiquetas, manteniendo la misma distancia para cada una, pero el primer y el ultimo elemento se encuentran pegados a los bordes.
  • space-evenly: Distribuye de manera equitativa todas las etiquetas, sin sumar las distancias entre elementos. si bien es muy practico, no es compatible con todos los navegadores

Organizacion de elementos (align-items)

  • flex-start: Define que todos los elementos se organicen desde el comienzo del flex (arriba) hasta el final, a diferencia de justify-content, no existe valor predeterminado
  • flex-end: Define que todos los elementos se organicen al final de la fila dentro del flex (abajo)
  • center: organiza los elementos de manera centrada sobre la fila (vertical)

Orientacion de los elementos (flex-direction)

  • row: es la direccion predeterminada de flexbox, distribuye los elementos en el eje horizontal y los organiza en el eje vertical.
  • column: en este valor los elementos se distribuyen en columna, es decir que justify-content funciona sobe el eje vertical y align-items sobre el eje horizontal
  • row-reverse: similar a row, la organizacion se mantiene en el eje vertical, pero el orden de la distribucion se invierte, los elementos se crean de derecha a izquierda.
  • col-reverse: similar a column, la distribucion se mantiene sobre el eje vertical, pero el orden de organizacion de invierte, los elementos se crea a abajo hacia arriba.

Introduccion CSS-Grid

Sistema de Grillas

  • grid-template-columns: Permite definir el tamaño y la cantidad de columnas que tendra nuestro sistema de grillas, por defecto el contenido se acomoda en una unica columna. Tambien se suele utilizar el valor auto para que las celdas se adapten a su contenedor
  • grid-template-rows: Permite definir el alto de las filas especificadas, por defecto las mismas poseen un valor auto, pero no determina la cantidad final de las mismas.
  • grid-gap: permite definir el espaciado entre celdas, indicando primero las filas
    • y luego las columnas. Este atributo tambien se puede utilizar por separado mediante row-gap y column-gap respectivamente.
  • grid-column: Indica las columnas ocupadas por la celda seleccionada, se expresa en forma de fraccion start/end; se compone de los atributos grid-column-start y grid-column-end
  • grid-row: Indica las filas ocupadas por la celda seleccionada, se expresa en forma de fraccion start/end; se compone de los atributos grid-row-start y grid-row-end

Organizacion y Distribucion del Contenido

  • Justify-content: Permite distribuir los elementos dentro de la etiqeuta contenedora sobre el eje horizontal, admite los mismos valores que flexbox.
    • flex-start: distribuye el contenido al inicio del flex (izquierda)
    • center: distribuye el contenido hacia el centro (horizontal)
    • flex-end: distribuye el contenido al final del flex (derecha)
    • space-around: Separa los elementos sumando las distancia entre etiquetas (mayor)
    • space-between: Separa los elementos de extremo a extremo, manteniendo la misma distancia entre etiquetas. (primer y ultimo elemento sin margen)
    • space-evenly: Distribuye equitativamente las etiquetas dentro del contenedor (no compatible con algunos navegadores)
    • baseline: Acomoda las etiquetas en base al tamaño de la tipografia
  • Align-content: Permite organizar los elementos dentro de la etiqeuta contenedora sobre el eje vertical, a diferencia de align-items que admite algunos valores, este atributo admite los mismos valores que justify-content.
    • flex-start: distribuye el contenido al inicio del flex (arriba)
    • center: distribuye el contenido hacia el centro (vertical)
    • flex-end: distribuye el contenido al final del flex (abajo)
    • space-around: Separa los elementos sumando las distancia entre etiquetas (mayor)
    • space-between: Separa los elementos de extremo a extremo, manteniendo la misma distancia entre etiquetas.(primer y ultimo elemento sin margen)
    • space-evenly: Distribuye equitativamente las etiquetas dentro del contenedor (no compatible con algunos los navegadores)
    • baseline: Acomoda las etiquetas en base al tamaño de la tipografia

Plantilla de Areas

  • grid-template-areas: Permite definir mediante un sistema de nombres separados por comillas, las areas abarcadas por las diferentes celdas de nuestro contenedor.
  • grid-area: Asigna el nombre de area a la celda seleccionada, haciendo que ocupe la posicion espeficada, indistintamente de su ubicacion actual.

PseudoClass y PseudoElement

  • Pseudoclass: Instancia o caracteristica de una etiqueta sobre la cual podemos trabajar para aplicar un estilo. Se caracterizan por llevar dos puntos ":" antes del nombre de la misma y usualmente se coloca pegado al elemento afectado.
    • :hover
    • :first-child
    • :last-child
    • :first-of-type
    • :last-of-type
    • :link
    • :visited
    • :active
    • :focus
    • :valid
    • :invalid
  • Pseudoelement: Parte o seccion dentro de una etiqueta que podemos utilizar para darle un estilo particular. Se caracterizan por llevar un doble dos puntos "::" antes del nombre, aunque se pueden utilizar con uno y tambien suelen colocarse pegadas al elemento en cuestion.
    • ::before
    • ::after
    • ::first-line
    • ::first-letter
