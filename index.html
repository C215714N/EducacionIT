Bootstrap

Responsive Web Design

Introduccion a Bootstrap

Es un Framework de CSS desarrollado por el equipo de twitter, que facilita la implementacion de un diseño web responsive. Utiliza la Metodologia BEM (Block Element Modifier) para las clases definidas en su hoja de estilos y las mismas se organizan segun su funcion en el sitio web. Los tipos de clases que podemos encontran se dividen en 3 categorias:

  • Helpers: clases que modifican alguna caracteristica de otro elemento previamente definido. Su objetivo es ofrecer valores alternativos.
  • Utilities: clases que afectan algun atributo en particular y suelen tener la regla !important al final del valor.
  • Components: clases que definen y se aplican a una estructura de elementos, aqui podemos ver reflejada la metodologia BEM (Block Element Modifier)

Caracteristicas Basicas

Como se menciona anteriormente, este framework utiliza la metodologia BEM que consiste en identificar los contenedores principales (Bloques) que definen a los componentes y a partir de ahi crear clases para cada Contenido (Elemento) que este relacionado con este. A la vez, tambien se pueden alterar los diseños predeterminados, definiendo una clase que indique el elemento afectado y su valor alternativo (Modificador)

Paleta de Colores

Podemos implementar un total de 8 colores diferentes, aunque tambien podemos crear nuestras propias clases utilizando las variables definidas en el selector :root de la hoja de estilos de Bootstrap. Al ser helpers, se aplican al atributo correspondiente, por ejemplo, si quiero cambiar el color del texto a rojo, debo agregar la clase text-danger al elemento que debemos modificar.

Unidades y Escalas

Cuando necesitamos aplicar las clase de utilities, debemos tener en cuenta los tamaños a implementar. Bootstrap como medida universal, ofrecer valores que van de 0 a 5 y cada uno corresponde a una medida que puede estar expresada en rem como el margin y el padding, o px como es el caso del border. Tambien debemos tener en cuenta que existen clases como h<n> y display que toman como referencia el ancho de la ventana (viewport) y cuyo tamaño decrece a medida que el ancho de la ventana disminuye.

  • m-1 y p-1 (.25rem)

  • m-4 y p-4 (1.5rem)
  • m-5 y p-5 (3rem)
  • border-0 (0px)
  • border-1 (1px)
  • border-2 (2px)

  • border-4 (4px)

Sistema de grillas

Una de las caracteristicas principales, por las cuales este framework se hizo popular, es su sistema de grillas flexible. Este utiliza 12 columnas basadas en porcentajes, que facilitan la distribucion de los elementos hijos. Para implementarlo es necesario utilizar la clase row que aplica un display flex y la clase col-n en los elementos hijos para definir el ancho del elemento en la grilla.

Puntos de Quiebre (Breakpoints)

Debido a que Bootstrap es un framework que se centra en el diseño responsive, es necesario definir los puntos de quiebre que nos permitiran modificar el diseño de nuestro sitio web segun el ancho de la ventana del navegador (viewport). Estos se implementan, al igual que el resto de clases, en la hoja de estilos y para utilizarlos simplemente debemos agregar las letras que corresponden al estilos que queremos aplicar.

class viewport max-width devices
<576px 100% smartwatch
sm ≥576px 540px smartphones
md ≥768px 720px tablets
lg ≥992px 960px netbooks
xl ≥1200px 1140px Desktop
xxl ≥1400px 1320px Smart Tv

En el caso de dispositivos pequeños, no es necesario utilizar alguna clase, ya que este framework esta pensado para diseñar nuestro proyecto a partir de la vista de dispositivo movil (mobile first). Por lo tanto, si no definimos algun punto de quiebre, el diseño se vera igual para todos los tamaños de ventana.

Cuando necesitamos realizar algun cambio, como la aparicion de elementos o diferenciar tamaños, podemos implementar las clases, mencionadas anteriormente, que afectan a la mayoria de elementos. Por ejemplo: en la tabla anterior, las celdas de la ultima columna tienen la clase d-none, pero si estas abriendo esto desde una pantalla que sea mayor a 576px aplica la clase d-sm-block, mostrando el contenido de la misma.
