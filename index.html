Elementos de una paginá Web

Estructura Semantica

Modelo implementado en HTML que permite definir la jerarquia y funcionalidad de cada parte de nuestra pagina, surge como una solucion al uso excesivo de DIVs, debido a que no existian elementos apropiados para el facilitar el posicionamiento.

  • HEADER cabecera de pagina, contiene los titulos principales y siempre debe estar al comienzo.
  • NAV navegacion de pagina, contiene los links mas importantes y puede estar dentro de la cabecera
  • MAIN contenido principal de toda la pagina, destacado para el SEO
  • SECTION seccion de la pagina o contenido contextualizado, no funciona fuera de la pagina
  • ARTICLE articulo de la pagina o contenido independiente, posee su propia estructura y funciona fuera del sitio.
  • ASIDE lateral o contenido tangencial que se relaciona indirectamente con sus hermanos, suele utilizarse para publicidad
  • FOOTER pie de pagina, debe ir al final y se utiliza para brindar informacion o un medio de contacto, copyright, un mapa del sitio y/o enlaces externos.

Atributos Principales

En una pagina web encontramos tres tipos de elementos, Etiquetas de Bloque, que se caracterizan por ocupar todo el ancho de su contenedor, generar salto de linea y cumplen la funcion de contenedores; Etiquetas de linea, son aquellas que comparten espacio con otros elementos, su tamaño es automatico y funcionan como contenido. Por ultimo, se encuentran las Etiquetas de linea-bloque que toman caracteristicas de las anteriores ya que se les puede dar tamaño, comparten espacio con otros elementos, pero se las trata como contenido(texto).

  • margin: distancia que existe entre etiquetas, no suma tamaño y siempre aplica el mayor.
  • border: linea que rodea la etiqueta, suma tamaño y por defecto su valor es 0.
  • padding: relleno de una etiqueta, podria considerarse sangria y al igual que el borde suma tamaño
  • content: elemento que se encuentran dentro de la etiqueta, se puede definir su tamaño.
					header{
						width: 350px;
						border: 5px solid #333;
						padding: 40px;
					}

Box-sizing

  • CONTENT-BOX: final-width = width + padding + border (440px)
    content-width = width (350px)
  • BORDER-BOX: final-width = width (350)
    content-width = width - padding - border (260px)

    Jerarquia de Selectores

    En css podemos indicar a que elementos les corresponde el conjunto de reglas que queremos aplicar, estos se dividen de manera jerarquia segun su importancia. Por defecto las etiquetas son las de menor jerarquia debido a que son selectores generales y los elementos que mayor peso tiene en las reglas son aquellos que se consideran unicos como los Id, ya que solamente puede correspoder uno por elemento.

    • Tag Selector
      • header{...}
      • header h1{...}
    • Class Selector
      • .header{...}
      • .header h1{...}
    • Id Selector
      • #header{...}
      • #header h1{...}

    Modelos de color

    CSS permite la integracion de diferentes esquemas de color, por un lado estan los colores HTML que se representan mediante un nombre y existen 140 valores como darkslateblue; por otro lado existe el modelo RGB o sintesis adhitiva, que utiliza los canales de color, en este caso hay 2 variantes, la decimal como rgb(68,68,136) y la hexadecimal como #444488. Por ultimo tambien encontramos el modelo Luz tono saturacion que se representa como hsl(245, 40%, 40%)

    Introduccion a Flexbox

    Atributo Justify-content

    Es un modelo flexible de representacion de objetos dentro de la pagina, que permite la implementacion de un eje principal para la distribucion de los elementos y un eje secundario para la organizacion de los mismos. Una vez aplicado, todos los elementos que sean hijos directos, se consideraran bloques, facilitando la aplicacion de estilos

    • flex-start: es el valor predeterminado, el cual dispone las etiquetas dentro del contenedor al principio del eje principal (por defecto: izquierda)
    • flex-end: este valor dispone los elementos del contenedor hacia el final del flex, manteniendo el orden de las etiquetas (por defecto: derecha)
    • center: este valor centra las etiquetas dentro de flexbox en base al eje principal (por defecto: horizontal)
    • space-around: separa las etiquetas aplicando la misma distancia para cada una, dichos valores se suman y como resultado la distancia de entre los border es menor a la que existe entre las etiquetas.
    • space-between: separa las etiquetas de extremo a extremo, quedando la primera y la ultima pegada a los bordes, pero la distancia que existe entre el resto es equitativa.
    • space-evenly: valor que no es compatible con todos los navegadores, pero que distribuye equitativamente todas las etiquetas dentro del contenedor

    Atributo Align-items

    En el eje principal podemos distribur los elementos a gusto, dentro de flexbox. Pero en el eje secundario podemos afectar la organizacion espacial de los mismos, por defecto este valor no esta definido y por lo tanto todas las etiquetas dentro del flex van a poseer la misma altura, debido a que se adaptan al contenedor.

    • flex-start: valor que dispone las etiquetas dentro del contenedor al principio del eje secundario (por defecto: arriba)
    • flex-end: este valor dispone los elementos del contenedor hacia el final del flex, manteniendo el orden de las etiquetas (por defecto: abajo)
    • center: este valor centra las etiquetas dentro de flexbox en base al eje principal (por defecto: vertical)
    • baseline: permite la organizacion dentro del eje secundario tomando en cuenta los tamaños de la tipografia en los diferentes elementos, para mantener la legibilidad
    • align-self: atributo que se define en una etiqueta que se encuentre dentro de flexbox y que permite alterar su organizacion con respecto al resto.

    Atributo flex-direction

    Como se menciona anteriormente, flexbox posee un eje principal y un eje secundario en lugar de llamarlos x y, esto se debe a que el funcionamiento de este modelo es unidimensional y la distribucion y organizacion de los contenidos se realiza en base a esto. Por lo tanto, podemos cambiar la orientacion de dichos y modificar la forma en la que se representan las etiquetas

    • row es el valor predeterminado de orientacion de ejes y funciona generando los contenido de izquierda a derecha (eje principal), de arriba hacia abajo (eje secundario)
    • column este valor genera los contenido en formato de columna, es decir, de arriba hacia abajo ( eje principal) y luego de izquierda a derecha (eje secundario) si existe un tamaño limite
    • row-reverse a diferencia del primero, en este caso se invierte la orientacion del eje principal, es decir que los elementos se crean de derecha a izquierda, alterando el orden de visualizacion.
    • column-reverse en este caso, se invierte la orientacion del eje principal, generando los contenido de abajo hacia arriba, pero manteniendo el eje secundario intacto

    Introduccion a CSS-Grid

    Implementacion del Sistema de Grillas

    Es un modelo de grillas multidimensional que nos permite organizar y distribuir los elementos dentro de un contenedor, pero a diferencia de Flexbox, CSS-Grid tiene sus ejes definidos, es decir, que la orientacion vertical y horizontal no se modifica. Pero aun asi podemos alterar el orden, la ubicacion y apariencia de los elementos que sean hijos directos.

    • grid-template-columns: Permite definir la cantidad de columnas que tendra nuestro sistema de grillas, a traves de sus tamaños, por defecto solo hay una.
    • grid-template-rows: Permite definir el tamaño de las filas indicadas, pero a diferencia de grid-template-columns, no podemos afectar la cantidad, por defecto es auto.
    • gap: separacion o calles entre etiquetas dentro de CSS-grid, funciona como shorthand de row-gap y column-gap para las filas y columnas respectivamente.
    • grid-column: Permite definir la ubicacion con respecto a las columnas, ademas del tamaño/cantidad de celdas que ocupara el elemento seleccionado dentro de CSS-grid
    • grid-row: Permite definir la ubicacion con respecto a las filas, ademas del tamaño/cantidad de celdas que ocupara el elemento seleccionado dentro de CSS-grid

    Organizacion y Distribucion de Contenido

    Al igual que flexbox, CSS-Grid nos permite acomodar los elementos dentro del contenedor, pero a diferencia de este, no se modifica la ubicacion dentro de los elementos de la tabla. Cabe destacar que este modelo divide el contenedor principal en partes mas pequeñas consideradas celdas, que funcionan como etiquetas.

    • flex-start: Coloca la tabla al principio del flex, izquierda para justify-content y arriba para Align-content.
    • flex-end: Coloca la tabla de al final del flex, derecha para justify-content y abajo para Align-content.
    • center: Dispone la tabla de al centro del flex, horizontal para justify-content y vertical para Align-content.
    • space-around: separa las celdas aplicando la misma distancia para cada una, dichos valores se suman y como resultado la distancia de entre los bordes es menor a la que existe entre las calles.
    • space-between: separa las celdas de extremo a extremo, quedando la primera y la ultima pegada a los bordes, pero el tamaño de las calles es equitativo.
    • space-evenly: valor que no es compatible con todos los navegadores, pero que distribuye equitativamente todas las celdas dentro del contenedor.

    Plantilla de Areas

    CSS-grid permite de manera sensilla definir nombres que luego seran asignados a los elementos que sean hijos directos dentro de un contenedor, organizando las celdas entre filas y columnas independientemente de su posicion original respecto al HTML.

    • grid-template-areas: Permite definir una platilla, utilizando nombres a modo de variables y luego indicando por filas, entre comillas, el lugar que ocupa cada celda.
    • grid-area: Asigna el area especificada al elemento seleccionado, respetando la plantilla del contenedor.
