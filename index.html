Introduccion a la Maquetacion

Elementos Semanticos

  • header representa la cabecera de pagina, donde colocamos el contenido introductorio. Entre las etiquetas que podemos encontrar en su interior estan los H1, algunos links e incluso la navegacion; aunque ocasionalmente tambien podemos apreciar alguna galeria. Esta etiqueta responde a la pregunta ¿quienes somos?
    • nav representa la navegacion principal, es donde debemos colocar los links mas importantes y por lo tanto, tenemos de definir cuales son. Existe una regla que determina "si para acceder a una pagina necesito hacer mas de 3 clicks, no merece la pena colocarla como parte de la navegacion principal" asi como tambien otra que dice "si todo es importante, nada es importante"
  • main representa al contenido principal de la pagina, toda informacion relevante se va a encontrar aqui segmentada en grupos segun el tipo de contenido, en caso de ser necesario. Aunque solamente con esta etiqueta es mas que suficiente para el indexado del contenido, dependiendo de la complejidad del mismo deberemos utilzar otras. Esta etiqueta responde a la pregunta ¿que le interesa al usuario?
    • article representa un articulo o contenido independiente dentro del sitio, es decir que funciona tanto dentro como fuera de la pagina, un ejemplo claro de este tipo de etiquetas son las noticias que poseen una estructura definida y se comprenden incluso fuera de su contexto original.
    • section representa una seccion o parte de un contenido dentro del contexto, el necesario que este dentro del sitio con los elementos con los que se relaciona porque al utilizarlo fuera de la pagina se pierde nocion del contenido, comunmente identificamos estas etiquetas por textos como "al igual que se menciona en el apartado anterior","se mencionara mas adelante" o dando por sentado ciertos temas.
    • aside representa el contenido lateral o tangencial, es decir, aquel que guarda una relacion indirecta con sus hermanos, normalmente se lo utiliza para publicidad o sugerencias dentro del mismo sitio y se le suele dar estilo de barra lateral, aunque al igual que el resto de las etiquetas semanticas, es de bloque.
  • footer representa el pie de pagina, aqui colocamos el contenido de despedida, como por ejemplo informacion del autor, copyright, redes sociales, mapa del sitio y datos o un formulario de contacto . El contenido de esta etiqueta es relevante para el usuario despues del contenido principal, engloba cualquier dato de interes y responde a la pregunta ¿que mas puede visitar?

Jerarquias de Selectores

  • tag selector selector menos especifico, por lo tanto con menor prioridad a la hora de aplicar las reglas, ya que apunta a todas las etiquetas (general). En el ejemplo se puede observar que se apunta a todas las etiquetas header y a todos los h1 que se encuentren dentro de header.
    • header{...}
    • header h1{..}
  • class selector selector con mayor especificidad, ya que apunta a todas las etiquetas que posean la clase seleccionada, indistintamente si estas pertenecen, o no, al mismo tipo (grupal). En este caso se apunta a todas las etiquetas con Clase header y a todos los h1 dentro de un elemento con clase header.
    • .header{...}
    • .header h1{...}
  • id selector selector mas especifico y por lo tanto con mayor prioridad, debido a que apunta a la etiqueta con el Id espeficado y como no debe existir mas de una con el mismo id es lo considera unico (individual). En esta situacion se apunta al elemento con id header y a todos los h1 dentro de dicha etiqueta.
    • #header{...}
    • #header h1{...}

Introduccion Atributos CSS

  • box-sizing: Indica al navegador que partes de las etiquetas se deben tomar en cuenta para calcular el tamaño final de las mismas, el valor predeterminado es content-box que contempla solamente el tamaño del contenido y luego debemos sumar el tamaño de padding y border para saber cual es el tamaño final de nuestro elemento. Por el contrario border-box contempla todo esto al definir el width.
    • Content-Box: content-width = width || final-width = width + padding + border
    • Border-Box: content-width = width - padding - border || final-width = width

Planificacion del Diseño

Unidades de Medida

  • Absolutas: son aquellas que, independientemente del tamaño de la pantalla donde se visualice el contenido, siempre representan lo mismo.
    • tamaños: cm - mm(0.1cm) - in(2.54cm) - 1pc (0.6in)
    • tipográficas: pt(1/72in - 0.35mm)
    • híbrida: px(depende de la resolucion de pantalla) - desktop 1px = 1px || phones 1px = 3px
  • Relativas: son aquellas que dependen exclusivamente de los valores que posea el elemento padre para determinar el aspecto de las etiquetas.
    • porcentaje: % de tamaño del elemento padre (contenedor)
    • tipográficas: em(ancho "M") - ex(alto "x") - ch(alto "0") - relativas al elemento padre
    • híbrida: rem(root em - depende del valor tipografico de HTML) - predeterminado = 16px
