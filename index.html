FrontEnd

Desarrollo del lado del cliente, que corresponde a toda caracteristica que puede interpretar/ejecutar el navegador sin intervencion del servidor, como por ejemplo El diseño UI/UX o algunas funcionalidades. Entre los lenguajes que componen el FrontEnd se encuentran los siguientes:

  • HTML
  • CSS
  • JS

HTML (HyperText Markup Language)

Lenguaje de marcado de hipertexto que utiliza etiquetas para identicar la funcion que cumple cada uno de los elementos de la pagina.
Las etiquetas poseen atributos que podemos indicar dentro de las mismas y cuando utilizamos las mismas dentro de la pagina, el orden de los factores altera el producto.

Existen diferentes tipo de etiquetas, la mayoria posee carga semantica, es decir, especifican el tipo de elemento que estamos estamos incorporando; pero existen algunas que se utilizan en funcion del diseño, a estas estas etiquetas se las conoce como presentacionales. Ambos grupos se pueden catalogar de la siguiente manera:

  • Etiquetas de Bloque son aquellas que ocupan todo el ancho de su contenedor, se les puede dar tamaño y generan salto de linea. Funcionan principalmente como contenedores.
  • Etiquetas de Linea son aquellas que comparten espacio con otras de su tipo, su tamaño es automatico y por defecto no se modifica. Funcionan principalmente como contenido

CSS (Cascading StyleSheets)

Las hojas de estilo en cascada corresponden a un lenguaje de diseño grafico, que utiliza selectores y reglas para definir la apariencia y representacion de las etiquetas que componen la pagina web.

En el caso de css, el orden de los factores no altera el producto, pero debemos tener en cuenta que el navegador dara mayor o menor prioridad al estilo segun como se implemente dentro de la pagina web.

  • en linea el codigo se escribe dentro de la etiqueta html utilizando el atributo style="...". Es el estilo con mayor prioridad, pero es mucho mas dificil de mantener.
  • en bloque el codigo se escribe separado del html, ya sea utilizando la etiqueta style o en un archivo css. Es mas facil de implementar, pero tiene menor prioridad comparado con el anterior.
