Introduccion a Bootstrap

Es un Framework de CSS desarrollado por el equipo de twitter, que facilita la implementacion de un dise単o web responsive. Utiliza la Metodologia BEM (Block Element Modifier) para las clases definidas en su hoja de estilos y las mismas se organizan segun su funcion en el sitio web. Los tipos de clases que podemos encontran se dividen en 3 categorias:

  • Helpers: clases que modifican alguna caracteristica de otro elemento previamente definido. Su objetivo es ofrecer valores alternativos.
  • Utilities: clases que afectan algun atributo en particular y suelen tener la regla !important al final del valor.
  • Components: clases que definen y se aplican a una estructura de elementos, aqui podemos ver reflejada la metodologia BEM (Block Element Modifier)

Caracteristicas Basicas

Como se comenta anteriormente, este framework utiliza la metodologia BEM que consiste en identificar los contenedores principales (Bloques) que definen a los componentes y a partir de ahi crear clases para cada Contenido (Elemento) que este relacionado con este. A la vez, tambien se pueden alterar los dise単os predeterminados, definiendo una clase que indique el elemento afectado y su valor alternativo (Modificador)

Paleta de Colores

Podemos implementar un total de 8 colores diferentes, aunque tambien podemos crear nuestras propias clases utilizando las variables definidas en el selector :root de la hoja de estilos de Bootstrap. Al ser helpers, se aplican al atributo correspondiente, por ejemplo, si quiero cambiar el color del texto a rojo, debo agregar la clase text-danger al elemento que debemos modificar.

  • bg-primary
  • bg-secondary
  • bg-light
  • bg-dark
  • bg-success
  • bg-warning
  • bg-danger
  • bg-info

Unidades y Escalas

Cuando necesitamos aplicar las clase de utilities, debemos tener en cuenta los tama単os a implementar. Bootstrap como medida universal, ofrecer valores que van de 0 a 5 y cada uno corresponde a una medida que puede estar expresada en rem como el margin y el padding, o px como es el caso del border. Tambien debemos tener en cuenta que existen clases como h<n> y display que toman como referencia el ancho de la ventana (viewport) y cuyo tama単o decrece a medida que el ancho de la ventana disminuye.

  • m-0 y p-0 (0 rem)
  • m-1 y p-1 (.25rem)
  • m-2 y p-2 (.5rem)
  • m-3 y p-3 (1rem)
  • m-4 y p-4 (1.5rem)
  • m-5 y p-5 (3rem)
  • border-0 (0px)
  • border-1 (1px)
  • border-2 (2px)
  • border-3 (3px)
  • border-4 (4px)
  • border-5 (5px)
