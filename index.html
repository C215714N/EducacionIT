Tema 1: Fluid Web Design

Utiliza porcentajes (%) y em (Ancho de la letra M) para definir los tamaños de elementos y tipografias respectivamente, si bien es funcional en pantallas medianas, en pantallas grandes o pequeñas se pierden algunos aspectos del diseño.
Una solucion al conflicto generado por los tamaños de pantalla fue lo que se conoce como Liquid Web Desing, el cual simplemente agrega limites a los tamaños que pueden adquirir los diferentes elementos mediante los attributos min-width:200px; y max-width:600px; aunque este centra la atencion en los espacios generados por la separacion del contenido.

Tema 2: Adaptive Web Design

Utiliza medidas estaticas (px) para generar versiones reducidas/resumidas del contenido utilizando puntos de quiebre segun el tamaño de pantalla. Uno de los mayores inconvenientes de este tipo de diseño fue que se comienza a partir de una Version de escritorio del contenido principal y por lo tanto a la hora de generar una version movil del mismo, se deben quitar elementos del diseño, reestructura todo el contenido e incluso diseñar nuevamente el sitio en una version simplificada. El problema de realizar este tipo de practicas es el contenido duplicado que se genera en el sitio, sin mencionar que dificulta el mantenimiento del mismo.

Tema 3: Responsive Web Design

Similar al Adaptive web design, utiliza breakpoints para los diferentes diseños, pero a diferencia de este implementa medidas relativas para que la estructura responda al tamaño de la pantalla. Este tipo de diseño comienza a implementarse a partir de una publicacion de Ethan Marcotte en el sitio A list apart donde menciona ciertos aspectos principales del diseño indicando las base de lo que hoy conocemos como Responsive Web Design.

Entre las caracteristicas principales encontramos la implementacion de un sistema de grillas, multimedia flexible y diseñar a partir de los dispositivos mas pequeños. Esta ultima caracteristica da nombre al diseño Mobile Firt, que consiste en la elaboracion del sitio comenzando por la vista movil y a partir de esta, mostrar y ocultar elementos a medida que vaya cambiando el tamaño de la pantalla.

Una ultima evolucion del diseño es la que se conoce como Content First, donde en lugar de generar manchas de texto se utiliza directamente la informacion final que sera publicada una vez subido el sitio al servidor. Esto se debe a que cuando utilizamos texto simulado (LOREM) no sabemos como funcionara el contenido hasta que coloquemos la informacion, y si bien utilizar los textos finales parecia mas complicado al comienzo, facilita el diseño final del proyecto porque tenemos una vista real de contenido .
