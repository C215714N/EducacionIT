Breakpoints

Una de las caracteristicas principales de este framework radica en el uso de puntos de quiebre de pantalla, es decir, unidades o tamaños en los que nuestros estilos pueden verse dispuestos a modificaciones. Para implementarlos debemos agregar a las clases las siglas que corresponde a dicho punto de quiebre, de lo contrario los estilos corresponden a todos los tamaños de pantalla.

Sistema de Grillas

Desde sus comienzos, bootstrap ofrece una reticula responsive de 12 columnas basadas en porcentajes, que se aplican sobre los elementos de la pagina utilizando las clases row para el contendor de nuestra grilla y col-n (n es un numero entre 1 y 12) para indicar la cantidad de columnas que debe ocupar un elemento, es decir, que porcentaje de su contenedor.

Si bien, podemos especificar un tamaño fijo para las columnas dentro de un contenedor, tambien podemos omitir esto utilizando la clase col e incluso adaptar la grilla utilizando los breakpoints previamente mencionados. Por ejemplo, si necesito que una columna ocupe la mitad del contenedor en pantallas medianas y un cuarto del mismo en pantallas extra grandes, utilizariamos las clases col-md-6 col-xl-3

Tipos de Diseño Web

Fluid Design

Utiliza porcentajes para definir el tamaño de los elementos, permitiendo que estos se adapten a su contenedor, pero en pantallas grandes los mismos se estiran demasiado. Tambien existen una variante llamada Liquid Design que establece el uso de maximos y minimos, pero esto genera espacios demasiado grandes entre elementos.

Adaptive Design

Consiste en diseñar una pagina para cada tipo dispositivo en el que se visualice el sitio, pero en este caso los contenidos se repiten y son independientes el uno del otro. Esta propuesta es ideal para clientes con bajo presupuesto, ya que generalmente se utilizan 2 versiones del sitio principal (desktop) y utilizan herramientas para el redireccionamiento.

Responsive Design

Diseño que utiliza multimedia flexible, un sistema de grillas adaptable e implementa el concepto one web for all para evitar la duplicidad de contenido. Este tipo de diseño debe ser planificado desde la vista mobile y de ser posible utilizando los contenido finales, para logar un mejor resultado a la hora de integrar los contenidos.
