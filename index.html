Tipos de Diseño

A lo largo de los años las estrategias de planificacion del diseño han ido evolucionando en funcion del publico y los dispositivos utilizados para la visualizacion de los sitios web. Si bien, en sus comienzos el uso de internet era algo reservado para unos cuantos y el acceso a las paginas web se obtenia a traves del uso de las computadoras de escritorio, hoy en dia podemos visitar cualquier sitio de internet desde una variada cantidad de dispositivos, cada uno con sus respectivos tamaños, como celulares, smartphones, tablets, laptops e incluso televisores.

Fluid Design

Modelo de maquetado que utiliza medidas relativas como porcentajes y em para definir el tamaño de los elementos dentro de la pagina. desde sus comienzos la web presenta este tipo de caractericas, pero uno de los problemas que se presenta con la misma es la incapacidad de poder predefinir un tamaño asi como en diseño grafico al utilizar un formato de papel.

Para solucionar el hecho que no se cuenta con un tamaño definido, se implemento el Liquid Design que consiste en agregar limites (max-width, min-width) a los elementos de la pagina, pero el problema que presenta esto son los espacios que se producen en dispositivos grandes y que los elementos se salgan del viewport en dispositivos pequeños.

Adaptive Design

Modelo de maquetado que plantea el diseño de contenido para cada tamaño de dispositivo al que nuestro sitio este destinado para su visualizacion, es decir, armar una pagina diferente para cada tamaño de pantalla y utilizando herramientas de redireccionamiento, que el navegador no ubique en el sitio adecuado.

El problema que surge con esto es la generacion de contenido duplicado, ya que utilizamos los mismos texto o diferentes imagenes para cada resolucion, sin tener en cuenta que esto afectara a los tiempos de carga del sitio. Esta opcion seria rentable para clientes con poco presupuesto debido a su facil mantenimiento.

Responsive Design

Modelo de maquetado cuyos origenes se corresponden a una publicacion realizada por Ethan Marcotte en el sitio A list Apart donde se plantea el concepto de One Web for All y comenta sobre diseñar para dispositivos moviles, la implementacion de media queries, sistemas de grillas para la planificacion, entre otras cosas.

Una de las caractericticas mas notables de este tipo de diseño radica en la planificacion desde dispositivos moviles (mobile first) para luego ir complejizando el mismo hasta cubrir los diferentes tamaños de pantalla y una de sus variantes se conoce como Content-First donde se hace incapie en trabajar con el contenido final.

Unidades Absolutas

  • distancia Unidades que poseen un valor definido, es decir que indistintamente del dispositivo, se representan siempre con el mismo tamaño y se utilzan para las cajas.
    • mm: milimetros
    • cm: centimetros
    • in: pulgadas (2.54cm)
  • tipografia Unidades con valores de referencia fijos que se utilizan para los texto, asi como tambien para los elementos que tengan relacion proporcional con estos.
    • pt: puntos (0.35mm)
    • pc: picas (12 puntos)
  • hibrida Unidades absolutas que toman como referencia un valor relativo como lo es la resolucion de pantalla, ya que si bien los pixeles poseen un tamaño, su representacion varias segun el valor de este factor.
    • px: pixel (1/96in)

Unidades Relativas

  • contenedor unidades cuyo tamaño depende de la caja contenedora, una vez obtenida esta unidad, el navegador la intepreta en px y se utiliza para definir el aspecto de los elementos.
    • % porcentaje (contenedor)
  • tipografia unidades que toman como referencia el tamaño de la tipografia del elemento contenedor, se utiliza para mantener las proporciones en las distancias y espacios de los contenidos.
    • em: ancho de "M"
    • ex: alto de "x"
    • ch: ancho del caracter "0"
    • %: porcentaje (contenido)
  • hibrida Unidades que toman como referencia un valor absoluto dentro de la pagina y a pesar de que este cambie dentro de la misma, siempre va a mantener su unidad.
    • rem: ancho de M en la Raiz (html)

Unidades Flexibles

  • viewport size Unidades que se calculan en base al tamaño de alguna de las dimensiones de la ventana donde se reproduce la pagina, normalmente se aplican para definir altos.
    • vw: % del ancho del viewport
    • vh: % del alto del viewport
  • viewport dimensions Unidades que toman de refencia el tamaño mas grande o chico del viewport, segun corresponde y se recalculan si redimensionamos la ventana del navegador, tambien se usan para tipografia flexible.
    • vmin: % del lado mas pequeño
    • vmax: % del lado mas grande

Boostrap

Framework de CSS que esta orientado al diseño responsive y esta compuesto por algunos archivos de CSS y JS. Su estructura de trabajo consiste en la aplicacion de clases predefinidas en las hojas de estilo mencionadas anteriormente para la aplicacion de estilos e implementacion de componentes, estos ultimos son basicamente elementos con su propia estructura definida por el equipo de desarrollo de bootstrap.

Breakpoints

Debido a que esta orientado al diseño responsive, Bootstrap posee estilos para los diferentes tamaños de pantalla y les asigna un nombre para facilitar su identificacion. ademas aporta clases para limitar el tamaño de los elementos, segun el tipo de dispositivo como se muestra a continuacion.

breakpoint screen-size max-width devices
xs < 576px 100% smartwatches
sm >=576px & <768px 540px smartphones
md >=768px & <992px 720px tablets
lg >=992px & <1200px 960px Laptops
xl >=1200px & <1400px 1140px Desktop
xxl >=1400 1320px HD Desktop

Implementacion de atributos

Cuando implementamos Bootstrap y a pesar de reinicio que este framework hace sobre nuestro diseño, necesitamos modificar el valor de atributos convencionales como margin, border o padding, simplemente debemos utilizar la clase con la inicial del atributo (en otros casos el nombre) y luego un guion seguido del valor de dicho atributo (esto varia segun el atributo en cuestion). Para las medidas Bootstrap suele implementar valores que van del 0 al 5, siendo variaciones de rem (3 = 1rem)

  • Margin
    • margin: .5rem; class="m-2"
    • margin-bottom: 1rem; class="mb-3"
    • margin-left: 1.5rem; class="ms-4"
    • margin: 1rem 0; class="my-3"
    • margin: .5rem 1rem .25rem; class="mt-2 mx-3 mb-1"
  • Border
    • border: 1px solid; class="border"
    • border-radius: .25rem; class="rounder"
    • border-color: #333; class="border-dark"(modifier)
    • border-left: 1px solid; class="border-start"
  • Padding
    • padding: .5rem; class="p-2"
    • padding-bottom: 1rem; class="pb-3"
    • padding-left: 1.5rem; class="ps-4"
    • padding: 1rem 0; class="py-3"
    • padding: .5rem 1rem .25rem; class="pt-2 px-3 pb-1"
