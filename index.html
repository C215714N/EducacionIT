Caracteristicas HTML5

Nuevos Elementos del estandar

Con el surgimiento del nuevo estandar, se incorporaron nuevas etiquetas que facilitaron a los desarrolladores la implementacion de estadisticas, contenido multimedia e incluso API's de dibujo. Si bien la mayoria de los navegadores actuales soporta dichas caracteristicas, debemos tener en cuenta que todavia existen algunos que no y por ende nuestro diseño debe contemplar dichas situaciones. Por eso podemos consultar sitios como Can I Use donde se nos mostrara una lista de compatibilidad de elementos.

Elementos Estadisticos

Cada vez que necesitemos comparar valores o evaluar una situacion especifica podemos utilizar 2 etiquetas, <progress> que debe implementarse cuando estamos llevando a cabo una tarea que aun no fue finalizada, ya que se utiliza para medir el avance dentro de la misma, ej: el progreso dentro de la capacitacion; y <meter> que debe implementarse cuando la tarea fue finalizada y debemos establecer una comparacion de valores, ej: el resultado de un examen.

  • Progress
    • value = undefined
    • value = 20; max = 100
    • value = javaScript
  • Meter
    • value < low < high < optimum
    • low < value < high < optimum
    • low < high < value < optimum

API's de Dibujo

Actualmente disponemos de 2 herramientas para dibujar dentro de nuestro documento HTML, <svg> (scalable vector graphics) que utiliza como lenguaje base XML y permite la creacion de graficos vectoriales mediante la implementacion de lineas y curvas; y por otro lado canvas (lienzo) que utiliza como lenguaje base javaScript y permite la creacion de graficos pixelares o tambien conocidos como mapa de bits. Si bien estas herramientas nos permiten realizar graficos dentro de nuestra pagina, tambien podemos utilizar nuestros conocimientos sobre las mismas, para poder editar graficos existentes.

  • SVG (Vectores)
  • Canvas (Raster)

Preprocesadores

Pseudolenguajes de CSS

Son herramientas que utilizan una sintaxis particular, tomando como base el lenguaje de hojas de estilos. Permiten la implementacion de todos los metodos y caracteristicas que existen en CSS, pero ademas agrega algunas funcionalidades extra; para utilizarlos debemos crear un archivo con el formato del preprocesador que vayamos a utilizar y luego generamos la hoja de estilos mediante el uso de un compilador, es decir, un interprete de las intrucciones previamente escritas en el codigo que las convierte a un archivo de CSS convencional.

LESS

Preprocesador que utiliza JavaScript como lenguaje compilador, tambien implementa varias caracteristicas del lenguaje como la asincronicidad. Para instalarlo debemos utilizar el siguiente comando con NodeJS npm i -g less y una vez que creamos la hoja de estilos la compilamos con el siguiente comando lessc 'input.less' 'output.less'.

Lamentablemente el preprocesador no posee un metodo de observacion, pero para ello instalamos otro paquete conocido como Less watch Compiler utilizando el comando con NodeJS npm i -g less-watch-compiler y para compilar y observar los cambios para actualizar las hojas de estilos utilizamos el comando less-watch-compiler 'input/' 'output/' 'input.less'

SASS

Preprocesador que utiliza Ruby como lenguaje compilador, a diferencia de LESS, todos sus metodos deben estar declarados en orden, ya que de lo contrario arrojara un error de compilacion, pero posee algunas caracteristicas automatizadas, como la generacion de bucles de distinto tipo y la distincion de metodos.s

Para instalarlo debemos ejecutar el comando con NodeJS npm i -g sass y para compilar nuestros archivos de manera automatizada, simplemente debemos ingresar el comando sass --watch input.scss input.css, cabe destacar que este preprocesador posee 2 tipos de sintaxis: SASS (formato original - sin scope) y SCSS (revision con implementacion de signos)
