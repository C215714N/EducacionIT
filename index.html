Desarrollo FrontEnd

Se conoce con este nombre al desarrollo web del lado del cliente (navegador), es decir, todas las caracteristicas y funcionalidades que este puede reproducir sin necesidad de consultar al servidor. Entre estas caracteristicas se encuentran (Diseño UI / UX / Logica de eventos).
Debemos tener en cuenta que el navegador es un programa complejo que recibe codigo y lo procesa para retornar un resultado y dentro de los lenguajes que puede interpretar, se encuentran los siguientes:

  • html
  • css
  • js

HTML (HyperText Markup Language)

El lenguaje de marcado de hipertexto utiliza etiquetas para definir la funcion que cumplen los elementos dentro de la pagina. En este caso el orden de los factores altera el producto, ya que el resultado de lo que interprete el navegador dependera de como sean leidas las lineas de codigo. Dentro de las etiquetas que podemos encontrar en una paginas, identificamos las siguientes:

  • De Bloque: Son aquellas etiquetas que ocupan todo el ancho de su contenedor, se les puede cambiar el tamaño y generan salto de linea, funcionan principalmente como contenedores.
  • De Linea: Son aquellas que ocupan el ancho de su contenido, comparten espacio con otros elementos de su tipo, su tamaño es automatico y no se puede cambiar, funcionan como contenido

CSS (Cascading StyleSheets)

Las hojas de estilo en cascada utilizan selectores y reglas para cambiar el aspecto de los elementos que se encuentran en la pagina web. En este caso el orden de los factores NO altera el producto, ya que es independiente a la disposicion de las etiquetas.

Dentro de las caracteristicas (atributos) mas comunes que podemos encontrar en las etiquetas, se encuentran los siguientes:

  • Margin: distancia que existe entre las etiquetas.
  • Border: linea que rodea a las etiquetas, por defecto es 0.
  • Padding: relleno que existe entre el borde y el contenido de la etiqueta.
  • Content: elementos que se encuentran dentro de la etiqueta, que podemos definirles un ancho y alto.
