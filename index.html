Tipos de Diseño

A lo de los años la forma de diseñar paginas web fue cambiando debido a las diferencias que existen entre el diseño estatico (papel) y el dinamico (web), debido a la gran cantidad de tamaños de pantalla (resolucion) en los que se puede visualizar nuestro sitio. En actualidad esta diferencia se aprecia mejor debido a que no solamente contamos con las computadoras, sino que las paginas tambien pueden ser visualizadas desde otros dispositivos como los son principalmente los celulares o las tablets.

Fluid Design

Modelo que utiliza porcentajes y em como unidades de medida principal para definir el tamaño de las etiquetas de la pagina permitiendo mantener la proporcion de los elementos, pero una de los principales problemas de este tipo de diseño se presenta cuando intentamos visualizar los contenidos en pantallas pequeñas o pantallas grandes, ya que el contenido se compacta demasiado o se estira de mas.

Una solucion para el problema que se genera con este tipo de diseño es el Liquid Web Design que basicamente consiste en la aplicacion de limites max-width min-width para evitar el estiramiento o compactacion del contenido. El inconveniente con esto esta en los espacios que se generan cuando limitamos el tamaño de los elementos, especialmente si distribuimos el contenido con porcentajes.

Adaptive Design

Modelo que consiste en la generacion de contenido por cada tamaño de dispositivo al que apunte nuestro diseño, normalmente se suelen utilizar herramientas de deteccion de pantalla para redirigirnos al sitio correcto en caso que visitemos la pagina desde un celular o tablet. Este diseño es facil de mantener ya que se realizan 2 o 3 archivos por cada pagina del sitio y se diseña a partir de la vista de escritorio.

Esta es una buena alternativa para clientes con bajo presupuesto, ya que su diseño es sencillo y el contenido esta optimizado para cada tamaño de pantalla al que apunta. El problema radica en la generacion de contenido duplicado, e incluso la carencia del mismo, ya que en pantallas pequeñas no suele haber la misma cantidad de informacion.

Responsive Design

Modelo cuyas bases se deben a una publicacion realiza por Ethan Marcotte en el sitio A List Apart en el año 2010 donde menciona algunos conceptos importantes para el diseño, como es el uso de un sistema de grillas para la distribucion del contenido, multimedia flexible para los diferentes tamaños de pantallas e incluso el uso de media queries para optimizar las caracterictiristicas del diseño.

Otra de las caracteristicas de este tipo de diseño es el Mobile First que consiste en la planificacion y diseño del sitio a partir de las vista mas pequeña (celular), ya que en su momento el uso de estos dispositivos para la visualizacion de paginas web iba en aumento. Existe otra implementacion conocida como Content First que plantea el uso de contenido final para una mejor apreciacion del diseño.

Unidades de Medida

Al momento de desarrollar un sitio web debemos tener en cuenta las medidas, proporciones y relaciones que existen entre los elementos nuestra pagina. Ya sea que planifiquemos el sitio desde cero o debamos realizar la codificacion de un mockup, es importante que al migrar el diseño se mantengan todas estas relaciones y se altere lo menos posible la proporcion cuando lo veamos en diferentes dispositivos.

Absolutas

Son unidades cuyo valor siempre es igual indistitamente del dispositivo donde se visualice, son ideales para definir limites de tamaño para la etiquetas como min-width o max-width, pero no se aconseja utilizarlas el atributos que tengan relacion con el texto como margin, padding o border. Tambien se suelen utilizar para definir el diseño del sitio para su impresion en papel.

  • distancia
    • mm milimetros
    • cm centimetros
    • in pulgadas (2.54cm)
  • tipografia
    • pc picas (1/6in - 4.23mm )
    • pt puntos (1/72in - 0.35mm )
  • hibrida
    • px pixel (1/96in - 0.2mm)

Relativas

Son unidades cuyo valor depende de otro elemento, generalmente el contenedor de la etiqueta afectada, se utilizan para definir el tamaño de los etiquetas y la tipografias implementadas dentro del sitio. Esto permite que ante un cambio la pagina mantenga sus proporciones, pero normalmente se aplican a los anchos ya que no se suele asignar una altura para los contenedores.

  • contenedor
    • % porcentaje del tamaño
    • fr fraccion (css-grid)
  • tipografia
    • % porcentaje de la tipografia
    • em ancho de "M"
    • ex alto de "x"
    • ch ancho del caracter "0"
  • hibrida
    • rem ancho de "M" de la raiz(HTML)

Flexibles

Son unidades relativas que toman como referente las dimensiones del viewport, es decir, la ventana donde se visualiza la pagina web y cuya informacion se obtiene mediante metadatos. Estas son ideales para definir alturas dentro del sitio y para implementar tipografia flexible o adaptable a cada tamaño de pantalla, siempre que se implementen maximos y minimos para la misma.

  • dimension
    • vw % del ancho del viewport
    • vh % del alto del viewport
  • proporcion
    • vmin % de la menor unidad del viewport
    • vmax % de la mayor unidad del viewport

Introduccion a Bootstrap

Es un framework de css desarrollado por el equipo de twitter cuya finalidad principal es facilitar el diseño responsive. Utiliza una metodologia de trabajo conocida como BEM (Block Element Modifier) donde se utilizan nombres de clase para identificar a los diferentes elementos de la pagina, dividiendolos en Bloques : contenedores de varios elementos y quienes definen el nombre inicial; Elementos: partes unicas dentro de un contenedor; y Modificadores: que son clases especiales que se utilizar para alterar alguna caracteristica de un bloque o elemento.

Atributos Principales

Una de las caracteristicas, bastante presente en cualquier framework de css, es el uso de clases de ayuda o helpers, que consisten en estilos individuales que se utilizan para definir el aspecto de los elementos cuando necesitamos cambiar algun atributo. En el caso de bootstrap suele utilizarse la letra del atributo y un valor que ronda entre 0 y 5, cuyas medidas estan establecidas en rem.

  • Margin
    • margin: .5rem; (.m-2)
    • margin-right: 3rem; (.me-5)
    • margin-top: .25rem; (.mt-1)
    • margin-bottom: 1rem; (.mb-3)
    • margin-left: 1rem; (.ms-3)
    • margin: 0 1.5rem; (.my-0 .mx-4)
  • Border
    • border: 4px solid; border-radius: .25rem (.border rounded)
    • border-right: 2px solid; (.border-end)
    • border-top: 3px solid; (.border-top)
    • border-bottom: 1px solid; (.border-bottom)
    • border-left: 5px solid; (.border-start)
    • border-radius: 50%; (.border .rounded-circle)
  • Padding
    • padding: .5rem; (.p-2)
    • padding-right: 3rem; (.pe-5)
    • padding-top: .25rem; (.pt-1)
    • padding-bottom: 1rem; (.pb-3)
    • padding-left: 1rem; (.ps-3)
    • padding: 0 1.5rem; (.py-0 .px-4)

Sistema de grillas

Una de las caracteristicas principales de este framework es su sistema de grillas responsive utilizando flexbox, el cual consiste en dividir un contenedor en 12 columnas, asignando un porcentaje a cada tamaño. Las clases se componen de las palabras col-n donde n representa el espacio ocupado por el elemento dentro de la grilla. Las columnas se acomodaran de izquierda a derecha siempre y cuando el contenedor posea la clase row Podemos indicar diferentes tamaños si agregamos puntos de quiebre a las mismas.

Puntos de quiebre

El sistema responsive funciona gracias a que las clases se organizan segun los diferentes puntos de quiebre, indentificando cada pantalla mediante su iniciales, los atributos por lo tanto, se utilizan de la siguiente manera attribute-breakpoint-value, tambien existe una clase llamada container para definir el tamaño maximo de un elemento y su contenido. Esta estructura de tamaños esta compuesta de la siguiente manera:

Breakpoint screen-size max-width Device
xs < 576px 100% Smartwatch
sm >= 576px && < 768px 540px Smartphones
md >= 768px && < 992px 720px Tablets
lg >= 992 && < 1200px 960px Laptops
xl >= 1200px && < 1400px 1140px Desktop
xxl >=1400px 1320px HD Desktop

Componentes

Otra de las caracteristicas principales que destaca a este framework es el uso de componentes, es decir, estilos predefinidos para cierto tipo de situacion o elementos de la pagina, independientemente de la etiqueta a la cual asignemos los mismos. Aqui es donde se nota mejor la estructura definida por el equipo de Bootstrap, ya que para que se apliquen los estilos correctamente, los elementos deben poseer las clases pertinentes.
