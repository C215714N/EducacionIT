Tipos de Diseño

A lo largo de los años la forma en la que se diseñan los sitios web ha ido evolucionando, debiendo adaptar la vista a los diferentes tamaños de pantalla. Esto se debe a que la web a diferencia de los medios fisicos no posee un tamaño definido, sino que existen variedad de resoluciones, en caso de las pantallas de escritorio y tamaños de dispositivo con sus respectivas densidades de pixeles.

Fluid Design

Este modelo utiliza porcentajes y ems para definir el tamaño de los elementos y la tipografia de la pagina, es el primer tipo de diseño que se implementa utilizando las caracteristicas nativas de las paginas web. El problema de este diseño radica en falta de escalabilidad para pantallas pequeñas o grandes, ya que los contenidos se compactan o se estiran demasiado.

Una solucion a este inconveniente es el Liquid Design que implementa el uso de maximos y minimos para limitar los tamaños de las cajas, pero al hacer esto los elementos se exponen a una separacion mayor en pantallas grandes o margenes compactos en pantallas pequeñas.

Adaptive Design

Este modelo se empezo a implementar a partir del aumento en la cantidad de dispositivos moviles como medio para la visualizacion de sitios web, y plantea el diseño de diferentes paginas segun el tamaño del dispositivo al cual apunte nuestro diseño. Tambien se implementan herramientas de deteccion de pantalla para redireccionar a la pagina correspondiente segun el tamaño del dispositivo.

El inconveniente es la cantidad de contenido duplicado que hay en el sitio, si bien los tiempos de carga estan optimizados, no se muestra la misma informacion en cada pagina; tambien es una alternativa para clientes con bajo presupuesto, que es facil de mantener.

Responsive Design

Este modelo surge a partir de una publicacion realizada por Ethan Marcotte en el sitio A List Apart donde plantea el uso de un sistema de grillas para la planificacion utilizando porcentajes, la implementacion de media queries y multimedia flexible, apuntando a diseñar a partir de la vista mobile en lugar de la vista de escritorio.

Una evolucion del diseño conocido como Mobile First, es el Content First que plantea la utilizacion de textos finales para la correcta visualizacion del sitio tal y como va a ser publicado, ya que al utilizar textos simulacion no tenemos una correcta nocion de como quedara el contenido una vez agregado a la pagina.

Unidades de medida

Al momento de realizar un sitio basado en una maqueta (mockup) debemos tener en cuenta las proporciones y los tamaños que posee cada elemento de la pagina, asi como tambien la tipografia implementada. Para esto vamos a utilizar diferentes tipos de unidades para los atributos de los elementos de la pagina.

Absolutas

Son aquellas que siempre representan lo mismo independientemente del tamaño de la ventana del navegador, ya que expresan un valor cuantificable. Se utilizan normalmente para definir limites, asi como tambien para los diseños de impresion.

  • distancia:
    • mm milimetros
    • cm centimetros
    • in pulgadas (2.54cm)
  • tipografia:
    • pc picas (1/6in 4.23mm)
    • pt puntos (1/72in 0.35mm)
  • hibrida:
    • px pixeles (1/96in 0.2mm)

Relativas

Son aquellas que dependen de otro elemento, normalmente su contenedor, para definir su tamaño y se utilizan principalmente para asignar el tamaño de los elementos de la pagina, asi como tambien a la tipografia.

  • contenedor:
    • % porcentaje
    • fr fraccion
  • tipografia
    • % tamaño de la fuente
    • em ancho de "M"
    • ex alto de "x"
    • ch ancho del caracter "0"
  • hibrida
    • rem ancho de "M" en la raiz(HTML)

Flexibles

Son aquellas que definen su tamaño en base a la ventana del donde se visualiza la pagina (viewport), se utilizan principalmente para la definicion de alturas, ya que rara vez se asigna este atributo a los elementos contenedores y tambien para la tipografia responsive.

  • tamaño
    • vh % del alto del viewport
    • vw % del ancho del viewport
  • relacion de aspecto
    • vmin % de la unidad mas pequeña del viewport
    • vmax % de la unidad mas grande del viewport

Bootstrap

Es un framework de CSS desarrollado por el equipo de trabajo de twitter que se caracteriza por la implementacion de clases orientadas al diseño responsive. Utiliza un sistema de grillas basado en porcentajes, clases de utilidad y componentes que aplicaremos sobre nuestro HTML.

Caracteristicas Basicas

Podemos modificar los elementos de nuestra pagina mediante la aplicacion de clases de utilidad, en la mayoria de los casos simplemente debemos colocar la inicial o nombre del atributo que queremos utilizar y un valor preestablecido, que suele rondar entre 0 y 5.

  • margin
    • margin: .5rem (.m-2)
    • margin-bottom:1rem; (.mb-3)
    • margin: .25rem 1.5rem (.my-1 .mx-4)
    • margin-right: 3rem (.me-5)
  • padding
    • padding: .5rem (.p-2)
    • padding-bottom:1rem; (.pb-3)
    • padding: .25rem 1.5rem (.py-1 .px-4)
    • padding-right: 3rem (.pe-5)
  • border
    • border-left: 4px solid (.border-start)
    • border-bottom: 3px solid; (.border-bottom)
    • border: 1px solid (.border)
    • border: 5px solid (.border-end)

Sistema de Grillas

Es una de las caracteristicas principales del framework que consiste en la implementacion de columnas (maximo 12) para definir el tamaño de los elementos de un contenedor, mediante la utilizacion de flexbox. Los tamaños se definen en porcentajes y para mantener las proporciones se utiliza una clase llamada container que limita el ancho del contenedor.
