Tipos de Diseño

Fluid Web Design

Modelo que utiliza solamente porcentajes para definir el tamaño de los elementos de la pagina. Este tipo de diseño se implemento desde el comienzo de las paginas web, pero su principal problema radica en que no se adapta bien a pantallas pequeñas o grandes, debido a que el contenido se ajusta proporcionalmente.

Una solucion a este modelo es el Liquid Design que implica el uso de limites para evitar que los elementos superen determinados tamaños o midan menos que tal valor. El problema que se genera con este tipo de diseño radica en los espacios resultantes.

Adaptive Web Design

Modelo que implementa el uso media queries o herramientas que detecten el tamaño de la ventana donde se visualiza y muestren el contenido en consecuencia. Implica la creacion de contenido por cada tamaño de pantalla objetivo, normalmente empezando desde la vista de escritorio, siendo la vista mobile una version reducida.

Este tipo de diseño suele ser ideal para clientes con poco presupuesto, ya que creamos una pagina o estilos por cada vista y por lo tanto son faciles de mantener, pero implica la duplicacion de contenido en el servidor, e incluso la eliminacion de elementos segun el tamaño de la pantalla.

Responsive Web Design

Modelo que surge gracias a una publicacion realizada por Ethan Marcotte en el sitio A List Apart donde menciona las capacidad que debe poseer un sitio web. En el mismo se menciona el uso de Media Queries, un sistema de grillas, La planicacion del diseño y el uso de medidas relativas

Podemos encontrar lo que se conoce como Mobile First y Content-First, que consiste en la planificacion y desarrollo a partir de la vista mobile, siendo la mas simple y la que nos permite visualizar como agregar los contenidos; mientras que el otro modelo varia solamente en la implementacion de textos finales, para una mejor nocion del trabajo realizado.

Contenido Multimedia

En caso que utilicemos el elemento video y definamos un width al mismo, el navegador mantendra las proporciones de este debido a que puede acceder a la informacion del mismo. Pero en caso de utilizar elementos que no pertenecen al sitio, como la etiqueta iframe que muestra contenido de manera asincrona, nuestro navegador solamente recibe la informacion enviada por nuestro servidor donde se aloja el sitio y muestra la informacion obtenida como si de una ventana se tratase.
