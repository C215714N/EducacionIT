Formularios HTML

Son herramientas que permiten el intercambio de datos entre cliente y servidor, para implementarlos dentro de una pagina web debemos colocar la etiqueta <form> y dentro de esta los elementos que forman parte de los mismos. Se debe tener en cuenta que solamente se utilizara para el envio los elementos que formen parte del formulario, es decir, que cualquier elemento fuera de la etiqueta sera ignorado.

¿Como funcionan?

A la etiqueta mencionada anteriormente, debemos colocarle por lo menos 2 atributos, estos son action="..." que indica lo que debe suceder tras realizar el envio, normalmente se utiliza un archivo en lenguaje de backend (ej: php); y method="..." que indica como se realizara el envio y quien tiene prioridad, entre los metodos mas comunes se encuentran los siguientes:

  • GET: metodo poco seguro ya que la informacion permanece visible en la barra de direcciones, pero cuya prioridad radica en la respuesta del servidor y por eso es mas rapido. Se utiliza en Buscadores
  • POST: metodo seguro que cifra los datos antes y despues del envio ya que en este caso la prioridad radica en la proteccion de los datos del usuario. Se utiliza para formularios de registro, incio de sesion y contacto.

¿Como se componen?

Dentro de un formulario podemos utilizar cualquiera de las etiquetas que existen, pero destacan 2 principalmente, los rotulos (LABEL) que se utilizan para identificar lo que debe Seleccionar/escribir el usuario y facilitan la accesibilidad; y los campos (INPUT / TEXTAREA / SELECT) que corresponden a los elementos donde el usuario podra seleccionar o escribir datos, estos deben poseer al menos los siguientes atributos:

  • name="..." valor con el que se identifica al campo o grupo de campos de formulario.
  • type="..." valor utilizado en los <input> que indica que tipo de dato es admitido por el control.
  • value="..." valor del campo que debe figurar en caso que el usuario seleccione alguna opcion.
